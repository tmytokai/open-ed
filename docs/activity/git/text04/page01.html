  2. 学習項目: [4] GitHub Pages

GitHub Pages は GitHub が提供している静的な Web ページのホスティングサービスです。
GitHub Pages を使うとHTMLファイルやMD(マークダウン)ファイルを使ってドキュメントを簡単に公開することが出来ます。
ただしPHPとかサーブレットとかを使ってバックエンドでプログラムを動かすことは出来ません。

では GitHub Page を使うために初期設定をしてみましょう。
今回は MD ファイルを使う方法を説明します。

Github にサインインする
ドキュメントを公開したいリポジトリを開くか、リポジトリを新規作成する
リポジトリの設定画面を開く
「Pages」 設定を開く
「Source」の「None」を「main (またはmaster)」にすると隣に「/(root)」と書いてあるフォルダ設定が現れる
フォルダ設定を適宜設定する。「/(root)」にした場合はリポジトリの先頭フォルダにHTMLファイルやMDファイルを置く。「/docs」にした場合は docs フォルダの中にHTMLファイルやMDファイルを置く
隣の「Save」ボタンを押すと「https://ユーザID.github.io/リポジトリ名/」にウェブページが公開される
フォルダ設定で設定した場所に index.md を作成する。Code 画面に戻り、Add file → Create new file を選択し、 index.md ( docs フォルダを設定した場合は docs/index.md ) とファイル名を指定する。次に下の編集画面に「こんにちは世界」と打ち込み、更に下の方にある「Commit」ボタンを押す

※ HTML ファイルを使いたい場合はファイル名を index.html にしてください

ページを開いて「こんにちは世界」と表示されていることを確認する
MDファイルを使う場合は Pages の設定画面の「Choose a thema」から任意のテーマを変更できる

※ テーマを変更してから実際に切り替わるまで若干時間がかかります
※ しばらく待ってもテーマが変わらなかったり、レイアウトが崩れる場合はページをスーパーリロードして下さい

pull して index.md をプロジェクトに取り込む(プロジェクトを新規作成した場合は git clone でクローンする)

あとは普通にローカルでエディタを MD ファイルを編集するだけです。
なおマークダウンのプレビュー機能がついているエディタを使って編集することがベターですが、GitHub の簡易エディタにもプレビュー機能が付いてますので、GitHub上で直接編集しても良いです。

MDファイルを編集したら、あとは コミット＆プッシュすれば内容が更新されます。
なおコミット＆プッシュしてから内容が更新されるまで少し時間がかかりますので、更新されなかった場合はしばらく待ってからもう一度ブラウザをリロードしてください。
