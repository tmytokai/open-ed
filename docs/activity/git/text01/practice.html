1. バージョン管理システムと git

演習 1-1 (チーム、10分): チームの各メンバーがこれまで行なってきたソースコードやドキュメント等のファイルのバージョン管理方法を話し合い、指定場所に 1 人 1 件名前付きで記述して下さい。


2. ローカルリポジトリとリモートリポジトリ

演習 1-2 (個人PC、20分): GitHub は代表的なリモートリポジトリのホスティングサービスです。演習で使用しますので GitHub に無料アカウントを作成しましょう。

IDとメールアドレスは任意です。
パスワードを忘れないように気をつけて下さい。
GitHub の無料アカウントで作れるリモートリポジトリの内容は全世界に公開されますので個人情報等はアップしないよう気をつけて下さい(※)。

※ 学生は申請すれば学生アカウントになって非公開リポジトリも作れるようになります。


3. git によるチーム開発の流れ

演習 1-3 (個人PC、30分): GitHub 上にテスト用のリモートリポジトリを作成し、commit と push の流れを試しましょう。

git を始めて使う場合は先に演習 2-1を実施して下さい。
GitHub にログインします。
「New Repository」ボタンを押します。
「Repository name」に「test」と入れます。
「Initialize this repository with a README」にチェックを入れます。
「Create Repository」ボタンを押すと test というリモートリポジトリが出来ます。
端末を開き、「git clone (リモートリポジトリのアドレス)」を実行するとリモートリポジトリをダウンロードしてローカルリポジトリを作成します。
ローカルリポジトリの中に cd で移動し、「README.md」というファイルが含まれている事を確認して下さい。
「hoge.txt」というテキストファイルを作成し(中の文字列は任意)、「git add hoge.txt」コマンドを実行して下さい(git add の意味は次の学習項目で説明します)。
「git commit -m "hoge.txt追加"」コマンドを実行し、ローカルリポジトリに対して hoge.txt の追加をコミットして下さい。
「git push」コマンドを実行し、リモートリポジトリに更新内容をアップロードして下さい。
GitHub に戻り、画面を更新して hoge.txt が追加されていることを確認して下さい。
