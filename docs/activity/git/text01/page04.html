git を利用するために「ワーキングツリー」「HEAD」「インデックス」という３つの用語も知っておく必要があります。


ワーキングツリー

「ワーキングツリー」はユーザーが作業中のフォルダのことで「ワークツリー」とも言います。

ワーキングツリー内にあるファイルやフォルダは git に管理させることができるので(※)、ファイルを更新したらリポジトリにコミットして保存することが出来ます。

※ ワーキングツリー内のどのファイルを git で管理させるかはユーザーが指定したり解除したり出来ます。


HEAD

「HEAD」はリポジトリの最新コミットを示す識別子のことです。

コミットを実行すると git は更新内容のスナップショット(※)を作成してリポジトリに保存します。

※ 一般的にはスナップショットの事も「コミット」と呼んでいます。

従って開発が進むにつれてコミットが複数出来るのですが、最新コミットのことを「HEAD」といいます。


インデックス

「インデックス」はコミットする前に更新内容の一時的な保存場所のことで「ステージ」ともいいます。
インデックスに更新内容を一時保存することを「インデックスに登録する」とか「add する」とか「ステージングする」とか言います。

インデックスに更新内容を一時保存しておけばいつでも保存時の内容にファイルやフォルダを復元できます。

なお、上で「コミットを実行すると更新内容のスナップショットを作成してリポジトリに保存する」と書きましたが、正確にはインデックスに保存された更新内容がリポジトリに保存されます。
