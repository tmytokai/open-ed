次に既存のファイルを更新してコミットしてみますが、基本的には新規作成と同じ様に add して commit という流れになります。
では実際にやってみましょう。

hoge.txtに b という文字を追加(改行は入れない)
hoge.txt を保存

この時点での「ワーキングツリー」「インデックス」「HEAD」は次の状態 6 になります。
ここではまだ「ワーキングツリー」しか更新されていません。

"hoge.txtを追加(S4)"

次に前のページで学んだ git add コマンドを使って hoge.txt をインデックスに登録します。

hoge.txt が更新されていることを確認: git status
git add hoge.txt
hoge.txt がインデックスに登録されているか確認: git status

この時点でワーキングツリーとインデックスが同じである状態 7 になりますが、まだコミットしていませんのでHEADはそのままです。

最後に git commit コマンドを使ってコミットします。

git commit -m "hoge.txtを更新(S8)"
コミットするファイルが無いか確認: git status
コミットログを確認: git log

するとインデックスの内容でスナップショットが作られてローカルリポジトリに登録され、そのスナップショットにHEADが移動します(状態 8 )。
