さてコミットするだけではあまり git を使う意味がありませんので、これからコミットを使ってファイルの更新差分を表示したり、ファイルの更新をキャンセルしたり、過去のファイル内容に復元するという操作をおこなって行きたいと思います。

ただしそれらの操作を実行するためにはどのコミットを対象にするか指定しなければいけませんので、まずはコミットの指定方法について説明します。

まず HEAD を使う方法から説明します。
HEAD は何回も言っている通りローカルリポジトリの最新コミットを示している識別子ですが、HEADの後に「~」(チルダ)や「^」(キャレット)(※)を追加することで過去のコミットを指定する識別子になります。

※ Windowsのコマンドプロンプトではキャレットはエスケープ文字になりますので、「^」は「^^」と2回入力する必要があります。

HEADでコミット指定: 
HEAD ・・・ 最新のコミット

HEAD~ または HEAD^ ・・・1つ前のコミット

HEAD~~ または HEAD^^ または HEAD~2 または HEAD^2 ・・・2つ前のコミット

HEAD~n または HEAD^n ・・・nつ前のコミット

なお「~」と「^」の意味は厳密には違うのですが(※)、とりあえず今回の演習では同じ意味であると考えて結構です。

※ git の「ブランチ機能」を使うと意味が変わり、HEAD^2 は 2 番目の親の意味になります。

例えば状態9のコミットにHEAD＋チルダ形式の識別子を付けると次のようになります。

さて、何個か前程度のコミットならチルダやキャレットを使って指定して良いのですが、かなり前のコミットだとHEADから数えて何個目になるのか分からなくなります。

その場合は git log コマンドで出てくるリビジョン番号(ハッシュ値とも言う)で指定します。

git log
日付やコメントで対象となるコミットを見つける
commit の後にある英数字の羅列がリビジョン番号(ハッシュ値)

さてこれで対象となるコミットの指定方法が分かりましたので、試しに演習で git show コマンドを使って過去のコミットで何をしたのか確かめてみましょう。
git show コマンドの構文は次の通りです。

指定したコミットの情報取得: 
git show <コミット>
