次にファイルの削除方法を学びます。

単にワーキングツリーからファイルを削除しても、そのファイルはインデックスには残り続けるので、何かのタイミングで消したはずのファイルが復活して大問題を起こしたりします。
そこで git で管理しているファイルは次の git rm コマンドを使ってワーキングツリーとインデックスから同時に消す必要があります。

ワーキングツリーとインデックスからファイルを同時に消す: 
git rm <ファイル>

ちなみにインデックスからは消したいけど、ワーキングツリーにはファイルを残したい場合(※)は --cached オプションを付けます。

※ 要するにあるファイルの git による管理を取りやめたい場合です。

インデックスからファイルを消す(＝ git による管理を止める): 
git rm --cached <ファイル>

さて、間違ってファイルを消してしまったとか言う場合はファイル内容の復元で学んだ checkout コマンドを使って復元できます。
ただしインデックスからも対象のファイルは消えてますので 

git checkout HEAD -- <ファイル名>

を使って HEAD から復元する必要があります。
