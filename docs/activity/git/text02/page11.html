  2. 学習項目: [2] ローカルリポジトリ

ローカルリポジトリの基本操作について学んできましたが、重要機能である「ブランチ機能」については取り扱いませんでしたので、ネット等を利用して自学自習して下さい。
git には他にも沢山便利な機能がありますので、これらについてもネット等を利用して自学自習して下さい。

ところで git を使っているとどのタイミングでコミットを実行すれば良いのか結構迷います。
ローカルリポジトリをいくら更新しても他の開発者に迷惑はかかりませんので、コミットを行うタイミングは原則自由です。例えば、

  • ある程度作業が進んだ時
  • ちょっと変なコードを試してみたい時 (一旦コミットしておけばいつでもHEADに戻れるので安心)
  • 休み時間の前
  • 何となく

など自分で適当にタイミングを決めて良いです。
ただし、1分おきとかソースを1行更新する毎とか、あまり頻繁にやり過ぎるとコミットログがとんでもないことになりますので、ほどほどの頻度を心がけて下さい。
逆に全くコミットしないのも、ゲームでセーブしないでボスに突撃するようなものですのでやめましょう。
