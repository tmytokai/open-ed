  2. 学習項目: [3] リモートリポジトリ

次にプッシュしてローカルリポジトリの更新内容をリモートリポジトリにアップロードして反映します。
ここではリモートリポジトリが更新されていない場合を試します。

この場合はgit によるチーム開発の流れで学んだ「衝突(conflict)」は生じませんので非常にシンプルです。


Aさんだけ作業するので、他のメンバーはAさんの作業を後から見る。
Aさんは pull を実行し、リモートリポジトリの更新内容をダウンロードしてローカルリポジトリに取り込む

git pull

この時点ではリモートリポジトリは更新されていないので「Already up-to-date」(既に最新版です)と表示される

Aさんは push を実行し、自分のローカルリポジトリの更新内容をリモートリポジトリにアップロードする

git push

※ GitHub のIDとパスワードを尋ねられます

全員がブラウザを開き、リモートリポジトリ「gitremote」の表示を更新し、hoge.txt の内容が変わっていることを確認する。

すると以下の状態3になります。
AさんのローカルリポジトリのHEADの内容がそのままリモートリポジトリにアップロードされている事が分かります。
