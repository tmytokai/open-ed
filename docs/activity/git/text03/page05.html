  2. 学習項目: [3] リモートリポジトリ

さてチーム開発をしていると他のメンバーがどんどんリモートリポジトリに更新内容を push してきますので、自分のローカルリポジトリとリモートリポジトリの内容がかけ離れていきます。

従って自分の作業が一区切りついて何か新しい作業を始める前にこまめに pull して、自分のローカルリポジトリとリモートリポジトリの内容の乖離をなるべく最小に保つように心がけると良いと思います(※)。
そうしないと、いざ pull する時に衝突が起こりまくって修正作業が大変になります。

※ ブランチ機能を使って効率よく並行作業する方法もありますが、このアクティビティではブランチ機能は取り扱いません。

ではAさん、Bさん、Cさんは pull を実行し、全員が同じ内容となっている状態7にして下さい。
