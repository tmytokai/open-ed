まとめ

リモートリポジトリのクローン

(1) 端末(※)を開いてホームディレクトリに移動する

※ Windows環境の場合は 端末 ＝ コマンドプロンプトと読み替えて下さい

(2)リモートリポジトリをクローンする

git clone <リモートリポジトリのアドレス>

(3) ls か dir でローカルリポジトリが出来ているか確認する

(4) ローカルリポジトリに cd する

・ 「origin」はリモートリポジトリの別名



ローカルリポジトリでの作業

・ 開発者間のローカルリポジトリは完全に独立している

・ 自分のローカルリポジトリ内だけで作業している限り、何をしても他の開発者に影響は無い



更新内容のプッシュ (衝突が無い場合)

(1) リモートリポジトリの更新内容をダウンロードしてローカルリポジトリに取り込む
(ただしこの時点ではリモートリポジトリは更新されていないので何も起きない)。

git pull

(2) ローカルリポジトリの更新内容をリモートリポジトリにアップロードする

git push

(3) 現在のローカルリポジトリの状態を確認する

git show

(4) 現在のリモートリポジトリの状態を確認する

git show origin

※ origin はリモートリポジトリの別名



更新内容のプッシュ (衝突がある場合)

(1) git pull して

CONFLICT (content)

と表示されたら衝突発生。

(2) git status でどのファイルで衝突が起きたか確認する。

「both modified」と表示されたファイルが衝突している。

(3) git diff で衝突内容を確認。

(4) 衝突したファイルを開く。

=======（マーカーという)よりも上の部分が自分がローカルリポジトリに対して行った更新内容、下がリモートリポジトリの更新内容。

(5) 内容を見ながらファイル修正。

(6) 修正したファイルを git add して衝突フラグを消す。

(7) git status で衝突フラグが消えている事を確認。

(8) git commit 実行

(9) 改めて git pull 実行。また衝突が発生したら (2) に戻る。

(10) 衝突が無くなったら git push 実行

・更新内容が被っていない時にプルすると git は自動的に中身を修正(マージと言います)し、ついでにコミットもしてくれる
