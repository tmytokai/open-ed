  2. 学習項目: [3] リモートリポジトリ

ここでは GitHub 上に作ったリモートリポジトリを使って多人数(今回はAさん、Bさん、Cさん、Dさんの4名)で共同開発をする演習を行います。

まずリモートリポジトリを新規作成する必要があるのですが、A さんに GitHub 上でリモートリポジトリを作成してもらい、そのリモートリポジトリをローカルリポジトリにクローンして使うことにします。

A さんは GitHub にログインし、「gitremote」という名前でリモートリポジトリを新規作成する。※ README の作成も忘れずに
A さんは GitHub の「gitremote」のページにある「Create New File」ボタンを押し、ファイル名に「hoge.txt」、中身に「name?」(改行無し)と入力し、「Commit new file」の欄に「init」というコメントを入力してコミットボタンを押す。
A さんは「hoge.txt」が「gitremote」に追加されている事を確認する。
A さんは「Setting」→「Collaborators」のページを開き、他のメンバーからIDを尋ねて検索し、「gitremote」の開発者としてメンバー登録する。
他のメンバーはブラウザで A さんの「gitremote」のページを開き、「Clone」ホダンを押して「gitremote」のアドレスをコピーする。
全員は端末(※)を開いて演習用のフォルダに移動する。

※ Windows環境の場合は 端末 ＝ コマンドプロンプトと読み替えて下さい

以下のコマンドでリモートリポジトリ「gitremote」を clone する

git clone アドレス

ls でローカルリポジトリが出来ているか確認する
ローカルリポジトリに cd する
ls で README と hoge.txt があることを確認する
cat で hoge.txt の中身を確認する。

以上のようにリモートリポジトリ「gitremote」をクローンすると以下の状態1になります。
リモートリポジトリの内容がA〜Dさんのローカル環境にそのままコビーされていることに注目して下さい。

なおリモートリポジトリのところにある「origin」とはリモートリポジトリの別名です。
