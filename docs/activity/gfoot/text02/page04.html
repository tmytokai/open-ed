1. スプリント プランニング

詳細設計:

このスプリントでは、「Actor」スーパークラスを継承した「FRUIT」サブクラスと、「FRUIT」を継承した「APPLE」と「ORANGE」サブクラスを作成します。

  • 「FRUIT」の画像は無しとします。
  • 「APPLE」と「ORANGE」の画像は任意とします。
  • 実行開始時に「APPLE」のオブジェクトを 5 つ作成してランダムに配置します。
  • 実行開始時に「ORANGE」のオブジェクトを 5 つ作成してランダムに配置します。

スプリント バックログの作成:

  1. 「FRUIT」「APPLE」「ORANGE」の作成
  2. 「MyWorld」 のコンストラクタのコーディング(APPLE、ORANGEオブジェクトの作成と配置)

クラス図の作成:

  • 開発作業を行う前にクラス図を描いて下さい。
  • クラス図「GF-2-2」をコピーし、ファイル名を「GF-2-3」に変更します。
  • 「FRUIT」「APPLE」「ORANGE」を追加します。
  • 「World」と「Actor」間の多重度を更新します。
    ひとつの「World (実体はMyWorld)」が 12 個の「Actor (TARO x1、HANAKO x1、APPLE x5、ORANGE x5)」を持っているので多重度は 1-12 です。

2. 開発作業

準備:

シナリオ「GF-2-2」をコピーし、シナリオ名を「GF-2-3」に変更します。
Greenfoot でシナリオ「GF-2-3」を開きます。


コーディング:

「Actor」スーパークラスを継承して「FRUIT」サブクラスを作成して下さい。 画像は無しです。また act メソッドは削除して下さい。
「FRUIT」を継承して「APPLE」「ORANGE」サブクラスを作成して下さい。 画像は任意とします。また act メソッドは削除して下さい。
「MyWorld」のコンストラクタで「APPLE」と「ORANGE」のオブジェクトをそれぞれ 5 つ作り、ランダムに配置して下さい。
x 座標の範囲は 0 〜600、y 座標の範囲は 0〜400 です。


テストとデバッグ:

Reset ボタンを押すと「APPLE」と「ORANGE」のオブジェクトがそれぞれ 5 つランダムに配置されることを確認して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は Reset ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のクラス図とソースコード(MyWorld)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
