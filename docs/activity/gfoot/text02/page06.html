1. スプリント プランニング

詳細設計:

このスプリントでは、「TARO」と「HANAKO」の動きを逆にします。

  • 「HUMAN」に public void move メソッドを追加し(※)、act メソッド内の移動処理を全てそこに移します。
  • 「HUMAN」の act メソッドから move メソッドを呼び出します。
  • 「HANAKO」で move メソッドをオーバーライドし、上下左右が逆向きの移動処理を書きます。
※ 本来なら public ではなくて protected にするか、抽象メソッド(純粋仮想関数)を使うか、又はデリゲーションを使った方が良いです。
それらについては自分で調べて下さい。

スプリント バックログの作成:

  1. 「HUMAN」 に move メソッドを追加し、移動処理を全てそこに移して act から呼び出す
  2. 「HANAKO」 で move メソッドをオーバライドしてコーディング(逆向きに移動)

クラス図の作成:

  • 開発作業を行う前にクラス図を描いて下さい。
  • クラス図「GF-2-4」をコピーし、ファイル名を「GF-2-5」に変更します。
  • 「HUMAN」と「HANAKO」に move メソッドを追加します。public であることに注意してください。

2. 開発作業

準備:

シナリオ「GF-2-4」をコピーし、シナリオ名を「GF-2-5」に変更します。
Greenfoot でシナリオ「GF-2-5」を開きます。


コーディング:

「HUMAN」クラスに public void move メソッドを追加して下さい。
act メソッドの移動処理部を切り取り、move メソッド内に貼り付けてください。
act メソッドから move メソッドを呼び出して下さい。
「HANAKO」に「HUMAN」の move メソッドを丸ごとコピーして貼り付けて下さい。
「HANAKO」の move メソッドを書き換えて上下左右の動きを逆にして下さい。


テストとデバッグ:

Reset ボタンを押して初期化して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は Reset ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のクラス図とソースコード(HUMANとHANAKO)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
