1. アクティビティ: Greenfoot によるオブジェクト指向開発入門
  2. 学習項目: [2] オブジェクト指向開発演習

演習時間: 15 分


1. スプリント プランニング

詳細設計:

このスプリントでは、全てのプレイヤーキャラクターの基となる「HUMAN」サブクラスを「Actor」スーパークラスを継承して作成します。
「HUMAN」のオブジェクトは矢印キーで上下左右に動かすことが出来ます。


スプリント バックログの作成:

  1. (必要なら)「MyWorld」の作成
  2. 「HUMAN」の作成
  3. 「HUMAN」 の act メソッドのコーディング

クラス図の作成:

開発作業を行う前にクラス図を描いて下さい。
ファイル名は「GF-2-1」とします。
World と MyWorld も忘れず描いて下さい。


2. 開発作業


準備:

Greenfoot を起動します。
新規 Java シナリオを作成します。シナリオ名は「GF-2-1」として下さい。


コーディング:

「World」スーパークラスを継承して「MyWorld」サブクラスを作成して下さい。

※ Greenfoot のバージョンによっては最初から MyWorld があります。その場合は作成しなくても良いです。

「Actor」スーパークラスを継承して「HUMAN」サブクラスを作成して下さい。 画像は割り当てないで下さい。
「HUMAN」サブクラスの act メソッド内に「矢印キーで上下左右に動かす」プログラムを記述して下さい。


テストとデバッグ:

「HUMAN」のオブジェクトをワールドにマウスで配置して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は動作を止めてデバッグして下さい。


リリース:

チーム内で答え合わせをし、提出用のクラス図とソースコード(HUMAN)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
