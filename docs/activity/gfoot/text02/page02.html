1. スプリント プランニング

詳細設計:

このスプリントでは、(必要なら)「MyWorld」を作ります。
また全てのプレイヤーキャラクターの基となる「HUMAN」サブクラスを「Actor」スーパークラスを継承して作成します。

  • 「MyWorld」の大きさは 600 x 400 とします。
  • 「MyWorld」の画像は任意とします。
  • 「HUMAN」の画像は無しとします。
  • 「HUMAN」はキーボードの矢印キーで上下左右に動かすことが出来ます。move量は 4 とします。

スプリント バックログの作成:

  1. (必要なら)「MyWorld」の作成
  2. 「HUMAN」の作成
  3. 「HUMAN」 の act メソッドのコーディング(オブジェクトを動かす)

クラス図の作成:

  • 開発作業を行う前にクラス図を描いて下さい。
  • ファイル名は「GF-2-1」とします。
  • World と MyWorld も忘れず描いて下さい。
  • Greenfoot の内部では「World」が「Actor」を集約しています(多分)。
    よって「World」と「Actor」を線で結び、中抜きのダイヤマークを「World」側に付けて下さい。
    多重度は省略して結構です(正確にはひとつの「World」が 0 個以上の「Actor」を持っているので、多重度は 1 - 0..* になります)。

2. 開発作業

準備:

Greenfoot を起動します。
新規 Java シナリオを作成します。シナリオ名は「GF-2-1」として下さい。


コーディング:

「World」スーパークラスを継承して「MyWorld」サブクラスを作成して下さい。 大きさは 600 x 400 とします。画像は任意とします。

※ Greenfoot のバージョンによっては最初から MyWorld があります。その場合は作成しなくても良いです。

「Actor」スーパークラスを継承して「HUMAN」サブクラスを作成して下さい。 画像は無しです。
「HUMAN」サブクラスの act メソッド内に「矢印キーで上下左右に動かす」プログラムを記述して下さい。 move 量は 4 です。


テストとデバッグ:

「HUMAN」のオブジェクトをワールドにマウスで配置して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は Reset ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のクラス図とソースコード(HUMAN)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
