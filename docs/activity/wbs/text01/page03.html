WBS の一番楽な作り方は作成ツールやテンプレートを利用することですが、今回は練習のために手作業で作成したいと思います。
なお今回は前のページで挙げた「親工程を左に置いて子工程を右に広げていく木構造 + ガントチャート」型の WBS を作成します。

まずは以下の手順でプロジェクトに含まれる工程を分解します。

  1. プロジェクトの最終目的を一番左(ルート)に書く
  2. ルートの工程をいくつかの子工程に分解し、ルートの右側に列挙してルートから線を引く
  3. 2で挙げた工程を更にいくつかの子工程に分解し、親工程の右側に列挙して親から線を引く
  4. 以上を繰り返す

この分解作業は全てのメンバーが具体的に何をすれば良いのか理解できる工程、つまり「アクティビティ」(またはワークパッケージ)が出てくるまで分解し続けます。
なお各階層の子工程の数は 3〜5 個位に留めておくのが良いらしいです。

さて必要なアクティビティを全て抽出したら、次はアクティビティ間の依存関係について考えます。
例えばアクティビティ A を開始するためには アクティビティB が終わってないといけないという状況は

B → A

の様に矢印を使って依存関係を表すことが出来ます。

最後にこの依存関係をもとにしてガントチャートを作ります。

典型的にはガントチャートでは最初の列にアクティビティ、次の列に担当者名や予算額などを記入し、各アクティビィティの開始時期から終了時期まで線を引いたり色を塗ったりします。
もちろんアクティビティ間には依存関係がありますので、依存しているアクティビティが終了するまで開始を待つ必要があります。
この様に依存関係に気をつけながら全てのアクティビティの開始時期と終了時期をスケジューリングします。

ところで適当にスケジューリングすると無駄な作業終了待ちが生じますので、可能な限りアクティビティを並列作業化して効率的にスケジューリングする必要があります。
ところが、どう頑張ってスケジューリングしても依存関係のせいで並列作業化出来ないアクティビティを繋げた経路(パス)が生まれます。
このパスのことを「クリティカルパス」と呼びます。
このクリティカルパス上のアクティビティの作業時間の合計がプロジェクトを完遂するための理論的な最小時間になります。
