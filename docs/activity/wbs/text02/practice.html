演習 2-1 (個人PC、40分): ひとりで家にいる時にお腹が空いたので何か食べることにしました。
しかし手元には水とカップラーメンくらいしかありません。いよいよお腹も空いてきました。
そこでカンバンを使って「ひとりで効率良くカップラーメンラーメンを作って食べる」ことにしました。
条件は以下のとおりです。

  • チームメンバー: 自分のみ
  • カップラーメン: 有り
  • お湯: 沸かしていない
  • 水・ヤカン・コンロ・箸: 有り

github にログインして適当なリポジトリを新規作成します。
「Projects」タブをクリックし、「Create a project」ボタンを押して「カップラーメン作成」という名前で新規プロジェクトを作成します。Project template は None のままで良いです。
テキストを参考に、「Add column」 ボタンを押して「ToDo」「Doing」「Done」の列を新規作成します。
カップラーメンを作って食べるために必要なタスクを自分で考えて、「ToDo」の列に登録して下さい(列の右上の＋ボタンで追加できます)。
一旦この状況でスクリーンショットを保存します。ファイル名は「KB-2-1-1」とします。
最初に実施したいタスクを選んでDoingに移します。
終了したタスクをDoneに移します。
全てのタスクを実施したらスクリーンショットを保存します。ファイル名は「KB-2-1-2」とします。
チーム内で答え合わせをし、提出用の図を選びます。
指定場所に提出用の「KB-2-1-1」「KB-2-1-2」を貼り付けて下さい。


演習 2-2 (チーム、60分): みんなで家で遊んでいた時にお腹が空いたので何か作って食べることにしました。
冷蔵庫を開けたらカレーの材料は揃っています。炊飯器にも朝炊いたお米が残ってました。
そこでカンバンを使って「チームで効率良くカレーを作って食べる」ことにしました。
条件は以下のとおりです。

  • チームメンバー: A、B、C、Dさんの 4 名
  • 材料(カレー粉、玉ねぎ、じゃがいも、人参、肉): 有り
  • お米: 既に炊いている
  • 水・鍋・コンロ・皿・スプーン: 有り

Aさんは github にログインして適当なリポジトリを新規作成し、残りの 3 名を Collaborator 登録します。
Aさんは「Projects」タブをクリックし「カレー作成」という名前で新規プロジェクトを作成します。
B〜Dさんも「カレー作成」プロジェクトを開きます。
Aさんはプロジェクト画面で「ToDo」「Doing」「Done」の列を新規作成します。
チームで話し合って「ToDo」にカレーを作って食べるために必要なタスクを登録して下さい。もし見辛かったらDnDで適当にタスクを並び替えて下さい。
一旦この状況で代表者はスクリーンショットを保存します。ファイル名は「KB-2-2-1」とします。
A〜Dさんは最初に実施したいタスクを選び、自分の名前(全世界に公開されるので本名は入れないこと)を記入してDoingに移します。
A さんはタスクを終了してDoneに移します(他の3名はそのまま)。
Aさんは次のタスクを選び、名前を付けてDoingに移します。ただし他の人のタスクが終わらないと次のタスクを選べないならその人のタスクがDoneになるまで待ちます。
同様に、Bさん→Cさん→Dさん→Aさんの順にDone、選択、Doingのステップを繰り返します。
全てのタスクを実施したら代表者はスクリーンショットを保存します。ファイル名は「KB-2-2-2」とします。
代表者は指定場所に「KB-2-2-1」「KB-2-2-2」を貼り付けて下さい。
