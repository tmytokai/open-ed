演習 2-1 (個人): ひとりで家にいる時にお腹が空いたので何か食べることにしました。
しかし手元には水とカップラーメンくらいしかありません。いよいよお腹も空いてきました。
そこでカンバンを使って「ひとりで効率良くカップラーメンラーメンを作って食べる」ことにしました。
条件は以下のとおりです。

  • チームメンバー: 自分のみ (1名)
  • 資金: 500 円
  • 食材: カップラーメン
  • 調味料: 無し
  • 水・鍋・ガス・コンロ・箸・どんぶり・その他調理器具: 有り

Github にログインして適当なリポジトリを新規作成します。
「Projects」タブをクリックし、「Create a project」ボタンを押して「カップラーメン作成」という名前で新規プロジェクトを作成します。Project template は None のままとします。
「Add a column」 ボタンを押して「ToDo」「Doing」「Done」の列を新規作成します。
カップラーメンを作って食べるために必要なタスクを WBS なども活用して考えて「ToDo」の列に登録して下さい。右上の＋ボタンを押すとタスクを追加できます。
一旦この状況でスクリーンショットを保存します。ファイル名は「KB_2_1_before」とします。
最初に実施したいタスクを選んで Doing に移します。
終了したタスクを Done に移します。
全てのタスクを実施したらスクリーンショットを保存します。ファイル名は「KB_2_1_after」とします。
指定場所に「実行前」「実行後」画像を貼り付けて下さい。


演習 2-2 (チーム): みんなで家で遊んでいた時にお腹が空いたので何か作って食べることにしました。
冷蔵庫を開けたらカレーの材料は揃っています。炊飯器にも朝炊いたお米が残ってました。
そこでカンバンを使って「チームで効率良くカレーを作って食べる」ことにしました。
条件は以下のとおりです。

  • チームメンバー: A、B、C、Dさん (4名)
  • 資金: 1000 円
  • 食材: カレー粉、玉ねぎ、じゃがいも、人参、肉、米(すでに炊いてある)
  • 調味料: 無し
  • 水・鍋・ガス・コンロ・箸・どんぶり・その他調理器具: 有り

(もしチームで利用するリポジトリが無かったら) Aさんは Github にログインして適当なリポジトリを新規作成し、残りの 3 名を Collaborator 登録します。
Aさんは「Projects」タブをクリックし「カレー作成」という名前で新規プロジェクトを作成します。Project template は None のままとします。
B〜Dさんも「カレー作成」プロジェクトを開きます。
Aさんは 「Add a column」 ボタンを押して「ToDo」「Doing」「Done」の列を新規作成します。
チームで話し合ってカレーを作って食べるために必要なタスクを WBS なども活用して考えて「ToDo」に登録して下さい。
一旦この状況で代表者はスクリーンショットを保存します。ファイル名は「KB_2_2_before」とします。
A〜Dさんは最初に実施したいタスクを選び、自分の名前(A〜D)を記入してDoingに移します。
A さんはタスクを終了してDoneに移します(他の3名はそのまま)。
Aさんは次のタスクを選び、名前を付けてDoingに移します。ただし他の人のタスクが終わらないと次のタスクを選べないならその人のタスクがDoneになるまで待ちます。
同様に、Bさん→Cさん→Dさん→Aさんの順にDone、選択、Doingのステップを繰り返します。
全てのタスクを実施したら代表者はスクリーンショットを保存します。ファイル名は「KB_2_2_after」とします。
代表者は指定場所に「実行前」「実行後」画像を貼り付けて下さい。
