カンバンの方式も色々あるのですが、今回は最もシンプルな「ToDo - Doing - Done」カンバン(※)について説明します。

※ Githubでは「Basic Kanban」と呼んでいます。

「ToDo - Doing - Done」カンバンの仕組みは非常に単純で、オフラインでもクラウドでも簡単に利用できます。
例えばオフラインの場合、壁や黒板やホワイトボードを用意してそれを看板に見立て、領域を横方向に 3 分割し、左の領域から上の方に

  • ToDo (やること)
  • Doing (作業中)
  • Done (済)

とタイトルを書き込みます

次に WBS の時と同じようにプロジェクトに含まれるタスクを抽出し、付箋やカード等に書き込みます。
なお、どの程度までタスクを細分化すれば良いのかについてはプロジェクトの規模や開発段階などによって変わってくるので一概には言えませんが、WBSの時と同じように「全てのプロジェクトメンバーが具体的に何をすれば良いのか理解できる」程度まで分解すれば良いかと思います。

さてタスクを抽出したら、それらを全て「ToDo」領域に貼り付けます。

次にタスク間の依存関係やメンバーの適性等を考慮しながら各メンバーが自分の担当するタスクを選び、自分の名前を書き込み、そのタスクを「Doing」領域に移動します。

あとは担当者はそのタスクを実行し、作業が完了したらタスクを「Done」領域に移動し、再び「ToDo」領域にあるタスクを選んで実行する(※)、というのを全てのタスクが「Done」状態になるまで繰り返すだけです。

※ なお依存関係のため次のタスクを選べず、待ち時間が生じる場合もあります

以上はオフラインでの話でしたが、クラウド上でも簡単に再現出来る仕組みであることが理解できると思います。
