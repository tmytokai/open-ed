1. スプリント プランニング

詳細設計:

このスプリントでは、「TARO」が「OBAKE」に触れたらメッセージを表示してゲームオーバーにします。
また「BALL」が「OBAKE」に触れたらメッセージを表示してゲームクリアにします。

  • Canvas オブジェクトの名前は「MyCanvas」とします
  • メッセージ表示用の Text オブジェクトの名前は「MessageText」とします
  • メッセージの内容、フォントサイズ、色等は任意とします。

スプリント バックログ:

  1. メッセージ表示用の「MessageText」の作成
  2. 「TARO」の当たり判定とゲームオーバー処理
  3. 「BALL」の当たり判定とゲームクリア処理

2. 開発作業

準備:

プロジェクト「UNITY-2」に対して作業を続けます。


スプリントバックログ 1: メッセージ表示用の「MessageText」の作成

「MyCanvas」という Canvas 型オブジェクトと「MessageText」という Text 型オブジェクトを新規作成します。
「MessageText」の位置やフォントサイズ等を適宜調整します。
調整が終わったら文字列(New Text)は消しておきます。


スプリントバックログ 2 : 「TARO」の当たり判定とゲームオーバー処理

「TARO」スクリプトをエディタで開き、 using 部に「using UnityEngine.UI;」を追加します。
OnTriggerEnter2D メソッド内に書いた処理の続きに、「TagObake というタグを持つオブジェクトと重なったら MessageText にゲームオーバーメッセージを表示してプログラムを停止する」プログラムを記述して下さい。


スプリントバックログ 3 : 「BALL」の当たり判定とゲームクリア処理

「BALL」スクリプトをエディタで開き、 using 部に「using UnityEngine.UI;」を追加します。
void Update() の下に void OnTriggerEnter2D( Collider2D collider ) メソッドを追加して下さい。
OnTriggerEnter2D メソッド内に、「TagObake というタグを持つオブジェクトと重なったら MessageText にクリアメッセージを表示してプログラムを停止する」プログラムを記述して下さい。


テストとデバッグ:

実行ボタンを押して動作確認して下さい。
上手く動作しなかった場合は実行ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のスクリプト(TARO.cs と BALL.cs)を選んで下さい。
指定場所に「ソースコード」を貼り付けて下さい。
