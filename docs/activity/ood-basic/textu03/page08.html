1. スプリント プランニング

詳細設計:

このスプリントでは、「OBAKE」オブジェクトを作って動かします。

  • 「OBAKE」 の画像は任意とします。
  • 「OBAKE」は「TARO」に向かって移動します。
  • 「OBAKE」は「BLOCK」をすり抜けます。
  • 移動速度(speed) は任意とします。
  • 「OBAKE」のタグは「TagObake」とします。

スプリント バックログ:

  1. 「OBAKE」オブジェクトの作成

2. 開発作業

準備:

プロジェクト「UNITY-2」に対して作業を続けます。


スプリントバックログ 1: 「TARO」オブジェクトの作成

画像を用意し「OBAKE」に名前変更して下さい(※ 拡張子は変更しないよう注意)。
画像を「OBAKE」オブジェクト化して下さい。サイズは適宜変更して下さい。位置は適当で結構です。なお「OBAKE」は物理演算エンジンで動かさないので Rigidbody 2D はアタッチしません。
「TARO」オブジェクトに Box Collider 2D をアタッチして下さい。なおis Trigger をチェックして下さい。衝突範囲は適宜調整して下さい。
「OBAKE」スクリプトを新規作成し、「OBAKE」オブジェクトにアタッチして下さい。
「OBAKE」スクリプトをエディタで開き、 void Update() の中に以下のプログラムを書いて下さい。
Vector2.MoveTowards メソッドを使って「OBAKE」が「TARO」に向かって移動する処理です。Vector2.MoveTowards については自分で調べて下さい。
speed は任意です。 

float speed = 任意;
transform.position = Vector2.MoveTowards( transform.position, GameObject.Find("TARO").transform.position, speed * Time.deltaTime);

スクリプトを保存して Unity エディタに戻って下さい。
「BLOCK」の手前に「OBAKE」を描画するため、「OBAKE」の Inspector ウィンドウの Sprite Renderer の Order in Layer に 1 をセットして下さい。
「OBAKE」のタグを「TagObake」として下さい。


テストとデバッグ:

実行ボタンを押して動作確認して下さい。
上手く動作しなかった場合は実行ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のスクリプト(OBAKE.cs)を選んで下さい。
指定場所に「ソースコード」を貼り付けて下さい。
