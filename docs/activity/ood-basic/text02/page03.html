1. 詳細設計

このスプリントでは、「HUMAN」クラスを継承してプレイヤーキャラクター「TARO」と「HANAKO」のクラスを作成します。
各クラスの詳細設計書は以下の通りです。

詳細設計:

  • 「TARO」クラスの画像は任意とします。
  • 「HANAKO」クラスの画像は任意とします。
  • 実行開始時に「MyWorld」 のコンストラクタで「TARO」オブジェクトをひとつ作成して左上に配置します。座標は (0,0)とします。
  • 実行開始時に「MyWorld」 のコンストラクタで「HANAKO」オブジェクトをひとつ作成して右下に配置します。座標は (600,400)とします。

2. 演習内容

クラス図の作成:

  • 開発作業を行う前にクラス図を描いて下さい。
  • クラス図「GF-2-1」をコピーし、ファイル名を「GF-2-2」に変更します。
  • 「TARO」と「HANAKO」を追加します。
  • 「World」と「Actor」間の多重度も追加して下さい。
    ※ ひとつの「World (実体はMyWorld)」がふたつの「Actor (実体はTAROとHANAKO)」を持っているので多重度は 1-2 です。

準備:

シナリオ「GF-2-1」をコピーし、シナリオ名を「GF-2-2」に変更します。
Greenfoot でシナリオ「GF-2-2」を開きます。


コーディング:

「HUMAN」クラスを継承して「TARO」「HANAKO」クラスを作成して下さい。 画像は任意とします。

※ そのままだと「HUMAN」の act メソッドがオーバーライドされてオブジェクトが動かなくなるので、「TARO」と「HANAKO」の act メソッドを消します。

「MyWorld」のコンストラクタで「TARO」と「HANAKO」のオブジェクトを作ってそれぞれ座標 (0,0) 、座標 (600,400) に動的配置します。


テストとデバッグ:

Reset ボタンを押すと「TARO」と「HANAKO」オブジェクトが左上と右下に表示されることを確認して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は Reset ボタンを押して動作を止めてデバッグして下さい。


提出:

チーム内で答え合わせをし、提出用のクラス図とソースコード(MyWorld)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
