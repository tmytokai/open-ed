オブジェクトの作成のページで書いたように、あるオブジェクトのコピーをひとつだけ作りたい場合は単に手動で Ctrl+c してから Ctrl+v するだけで OK です。
しかし弾丸みたいに大量のオブジェクトが必要な時にいちいち手動でコピーしていては大変です。

あるオブジェクトを大量にコピーして画面に表示させたい場合、Java や C++ の様なクラスベースのオブジェクト指向言語ならキャラクターの元となるクラスを書いてから new を繰り返してオブジェクトを作るだけで良いのですが、Unity はプロトタイプベースの開発環境なので「Prefab(プレハブ)」機能という少し変わった方法を用います。

この Prefab 機能は元となるオブジェクトから Prefab という種類のアセットを作り、プログラム実行時に Prefab をインスタンス化して元のオブジェクトのコピーを作るという機能です。
と口でだけ説明しても良く分からないと思いますので、具体的なやり方をこれから説明していきます。

まず元となるオブジェクトを Prefab 化します。

元となるオブジェクトを Prefab 化する

Hierarchy ウィンドウ上で Prefab 化したいオブジェクトを選択する。
Assets フォルダに選択したオブジェクトを DnD すると同じ名前の Prefab が出来る。
Hierarchy ウィンドウ上で Prefab 化済みのオブジェクトを削除する。

この Prefab をあるスクリプトの中でインスタンス化するためには次の Instantiate メソッドを使います。

Prefab をインスタンス化する 
public static Object Instantiate( prefab, 表示座標, 回転角度 );

戻り値(Object): インスタンス化したオブジェクト

prefab: インスタンス化したい Prefab
表示座標: 下のソース 1 参照
回転角度: 下のソース 1 参照

例えば次のスクリプト HOGE は prefab を Start メソッドの中で座標 (0.5, -0.1)にインスタンス化する例です。

ソース 1: Prefab を Start の中で座標 (0.5, -0.1) にインスタンス化するスクリプト 
public class HOGE : MonoBehaviour {

	public GameObject prefab; // ※1

	void Start () {
		Vector2 position = new Vector2 (0.5f, -0.1f);  // Vector2 型で座標指定
		Quaternion rotation = Quaternion.identity; // Quaternion 型で回転角度指定。Quaternion.identity は無回転の意味
		Instantiate (prefab, position, rotation );                
	}
}

ただし、このスクリプト HOGE を他のオブジェクトにアタッチしてプログラムを実行しても public GameObject prefab フィールド(※1の行)が空のままなので何も起きません。

prefab フィールドにインスタンス化したい Prefab をセットするには以下のようにします。

prefab フィールドにインスタンス化したい Prefab をセットする

スクリプトを保存して Unity エディタに戻る。
Hierarchy ウィンドウ上で HOGE スクリプトをアタッチしたオブジェクトを選択する。
Inspector ウィンドウの HOGE(Script) コンポーネントに「prefab」というプロパティ(こういうの)が出来ているのを確認する。
Assets フォルダ上でインスタンス化したい Prefab を選択して、(3)の None と書いてあるエリアに DnD する。
ちなみに複数の Prefab をインスタンス化したい場合は、例えば prefab1、prefab2、・・・みたいに public GameObject 型のフィールドを複数用意する。

後は実行すれば Scene ビューに表示されていないオブジェクトが実行時にインスタンス化され表示されていると思います。
