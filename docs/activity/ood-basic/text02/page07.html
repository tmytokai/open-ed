1. 詳細設計

この工程では、ゲームオーバーとゲームクリアの処理を作ります。
「TARO」オブジェクトが果物に触れたらゲームオーバーです。
「HANAKO」オブジェクトが果物を全て食べたらクリアです。
そのため「HUMAN」「TARO」クラスを以下のように拡張します。

詳細設計:

  • 「HUMAN」クラスに public void eat メソッドを追加し、act メソッド内の当たり判定処理を全てそこに移します。
  • 「HUMAN」クラスの act メソッド内から追加した eat メソッドを呼び出します。
  • 「HUMAN」クラスの eat メソッドで果物を 10 個食べたらメッセージを表示してゲームを停止します。メッセージ内容と位置は任意です。
  • 「TARO」クラスで eat メソッドをオーバーライドし、果物に触れたらメッセージを表示してゲームを停止します。メッセージ内容と位置は任意です。

2. 演習内容


クラス図の作成:

  • 開発作業を行う前にクラス図を描いて下さい。
  • クラス図「GF-2-5」をコピーし、ファイル名を「GF-2-6」に変更します。
  • 「HUMAN」と「TARO」に eat メソッドを追加します。public であることに注意してください。

準備:

シナリオ「GF-2-5」をコピーし、シナリオ名を「GF-2-6」に変更します。
Greenfoot でシナリオ「GF-2-6」を開きます。


コーディング:

「HUMAN」クラスに public void eat メソッドを追加して下さい。
act メソッドの当たり判定処理部を切り取り、eat メソッド内に貼り付けてください。
act メソッドから eat メソッドを呼び出して下さい。
eat メソッドに、fruits フィールドの値が 10 になったらクリアメッセージを出してプログラムを停止する処理を追加して下さい。メッセージ内容と位置は任意とします。
「TARO」クラスに「HUMAN」クラスの eat メソッドを丸ごとコピーして貼り付けて下さい。
「TARO」クラスの eat メソッドを書き換えて、果物に触れたら即ゲームオーバーメッセージを出してプログラムを停止する処理を追加して下さい。メッセージ内容と位置は任意とします。


テストとデバッグ:

Reset ボタンを押して初期化して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は Reset ボタンを押して動作を止めてデバッグして下さい。


提出:

チーム内で答え合わせをし、提出用のクラス図とソースコード(HUMANとTARO)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
