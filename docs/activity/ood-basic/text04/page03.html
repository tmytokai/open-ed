1. スプリント プランニング

詳細設計:

このスプリントでは、「TARO」オブジェクトを作って上下左右に動かします。

  • TARO の画像は任意とします。
  • 移動速度(speed) は任意とします。

また TARO が画面外に出られないように 4 方向に見えない壁オブジェクトを作って置き、TARO を画面内に閉じ込めます。

  • 壁オブジェクトの名前は「top」「bottom」「left」「right」とします。

スプリント バックログ:

  1. 「TARO」オブジェクトの作成
  2. 4方向の壁オブジェクトの作成

2. 開発作業

準備:

プロジェクト「UNITY-2」に対して作業を続けます。


スプリントバックログ 1: 「TARO」オブジェクトの作成

画像を用意し「TARO」に名前変更して下さい(※ 拡張子は変更しないよう注意)。
画像を「TARO」オブジェクト化して下さい。サイズは適宜変更して下さい。位置は左上の適当な位置で結構です。
「TARO」オブジェクトに Rigidbody 2D をアタッチして重力(Gravity Scale)を 0 にし、Constraints → Freeze Rotation → Z にチェックを入れて下さい。
「TARO」オブジェクトに Box Collider 2D をアタッチして下さい。衝突範囲は適宜調整して下さい。
「TARO」スクリプトを新規作成し、「TARO」オブジェクトにアタッチして下さい。
「TARO」スクリプトをエディタで開き、 void Update() の下に void FixedUpdate() メソッドを追加し、FixedUpdate メソッド内に「方向ボタンで上下左右に動かす」プログラムを記述して下さい。 speed は任意です。


スプリントバックログ 2 : 4方向の壁オブジェクトの作成

まず底から作ります。Hierarchy ウィンドウで右クリック → Create Empty で空のオブジェクトを作り、「bottom」に名前を変更して下さい。
「bottom」オブジェクトに Box Collider 2D をアタッチし、衝突範囲を表示画面いっぱいに引き伸ばしてください。
「bottom」オブジェクトを画面外の底に移動して下さい。
コピーするなりして「top」「left」「right」オブジェクトも作成して下さい。


テストとデバッグ:

実行ボタンを押して動作確認して下さい。
上手く動作しなかった場合は実行ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のスクリプト(TARO.cs)を選んで下さい。
指定場所に「ソースコード」を貼り付けて下さい。
