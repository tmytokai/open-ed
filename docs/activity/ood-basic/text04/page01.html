ここでは別のミニゲームを開発してみましょう。

1. 要求定義

要求仕様書は以下の通りです。

要求仕様書

名称TARO と OBAKE
用途・目的暇つぶし用ミニゲーム
対象ユーザ6 歳以上の男女
システム構成 OS: Windows 又は Linux
PC: 低スペック
開発環境: Unity2D
概要 ・PC 上で動く 2 次元アクションゲーム
・キーボードのみで操作
・画面上に TARO というプレイヤーキャラと OBAKE という敵キャラと BLOCK という複数の障害物を表示する
・重力により TARO は下に落ち続ける
・ユーザーはキーボード操作により TARO を左右に動かしたりジャンプさせたり出来る
・TARO は BLOCK をすり抜けられない。
・TARO はキーボード操作により BALL という弾を投げられる
・BALL は最初に投げた方向に動き続ける
・OBAKE は動きまわる
・BALL が OBAKE に触れたらゲームクリア
・TARO が OBAKE に触れたらゲームオーバー


2. 全体仕様書の作成

ゲーム仕様は以下のように策定しました。

ゲーム仕様

・背景(MyWorld と名付ける)は一枚絵とし、スクロールしない
・TARO(x1)、OBAKE(x1)、BLOCK(x複数) は適当に配置する
・重力により TARO は下に落ち続ける
・TARO は左右のカーソルキーで左右に動かせる
・TARO は z キーでジャンプする
・TAROジャンプ中でも左右移動可能
・TARO は x キーで向いている方向(左右)に BALL を投げる
・TARO は BLOCK をすり抜けられない
・BALL は複数個投げられる
・BALL は動いている途中に方向とスピードを変更しない
・BALL は BLOCK をすり抜けられる
・OBAKE は適当なアルゴリズムに従って動く
・OBAKE は BLOCK をすり抜けられる
・BALL が OBAKE に触れたらゲームクリアとなりメッセージ表示
・TARO が OBAKE に触れたらゲームオーバーとなりメッセージ表示

画面仕様図は以下の通りです。

画面仕様図

クラス設計は以下の通りです。

クラス設計

クラス名内容
MyWorld ・背景クラス
・特に何もしない
BLOCK ・障害物クラス
・特に何もしない
TARO ・プレイヤーキャラクラス
・キーボード操作(左右移動)を行う
・キーボード操作(ジャンプ)を行う
・BALL を投げる
・OBAKE に触れたか判定してゲームオーバー処理を行う
BALL ・弾クラス
・最初に投げた方向に動き続ける
・OBAKE に触れたか判定してゲームクリア処理を行う
OBAKE ・敵クラス
・適当に動く

継承などは特に使っていないので、今回はクラス図は省略します。
