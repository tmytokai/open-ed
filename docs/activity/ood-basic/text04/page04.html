1. スプリント プランニング

詳細設計:

このスプリントでは、「HANAKO」オブジェクトを作って上下左右に動かします(ただし動きは反転)。

  • HANAKO の画像は任意とします。
  • 移動速度(speed) は TARO と同じとします。

また TARO とはスレ違い可能にし壁オブジェクトはすり抜けられないようにレイヤーを設定します。

  • TARO と HANAKO のレイヤーは「LayerPlayer」とします。

スプリント バックログ:

  1. 「HANAKO」オブジェクトの作成
  2. レイヤーの設定

2. 開発作業

準備:

プロジェクト「UNITY-2」に対して作業を続けます。


スプリントバックログ 1: 「HANAKO」オブジェクトの作成

画像を用意し「HANAKO」に名前変更して下さい(※ 拡張子は変更しないよう注意)。
画像を「HANAKO」オブジェクト化して下さい。サイズは適宜変更して下さい。位置は適当で結構です。
「HANAKO」オブジェクトに Rigidbody 2D をアタッチして重力(Gravity Scale)を 0 にし、Constraints → Freeze Rotation → Z にチェックを入れて下さい。
「HANAKO」オブジェクトに Box Collider 2D をアタッチして下さい。衝突範囲は適宜調整して下さい。
「HANAKO」スクリプトを新規作成し、「HANAKO」オブジェクトにアタッチして下さい。
「HANAKO」スクリプトをエディタで開き、 void Update() の下に void FixedUpdate() メソッドを追加し、FixedUpdate メソッド内に「方向ボタンで上下左右に動かす。ただし上を押したら下、下を押したら上、左を押したら右、右を押したら左に動く」プログラムを記述して下さい。 speed は TARO と合わせて下さい。


スプリントバックログ 2 : レイヤーの設定

「TARO」、「HANAKO」オブジェクトのレイヤーを「LayerPlayer」にしてください。
Layer Collision Matrix の設定を開き、LayerPlayer - LayerPlayer 間のチェックを外して下さい。


テストとデバッグ:

実行ボタンを押して動作確認して下さい。
上手く動作しなかった場合は実行ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のスクリプト(TARO.cs)を選んで下さい。
指定場所に「ソースコード」を貼り付けて下さい。
