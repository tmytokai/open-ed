1. スプリント プランニング

詳細設計:

このスプリントでは、タイムアップ処理を作ります。

  • 「MyWorld」クラスに private int rtime フィールドを追加します。rtime は残り時間を表す整数です。
  • rtime の初期値は 1000 です。
  • 「MyWorld」クラスに public void act メソッドを追加し、act メソッドで rtime をデクリメントして画面に表示します。表示位置は左下なら任意の場所でOKです。
  • rtime が 0 以下になったらタイムアウトメッセージを表示してゲームを停止します。メッセージ内容と表示位置は任意です。

スプリント バックログ:

  1. 「MyWorld」クラスのコンストラクタのコーディング( rtime フィールドの初期化)
  2. 「MyWorld」クラスの act メソッドのコーディング( rtime のデクリメントと表示とタイムアウト処理)

クラス図の作成:

  • 開発作業を行う前にクラス図を描いて下さい。
  • クラス図「GF-2-6」をコピーし、ファイル名を「GF-2-7」に変更します。
  • 「MyWorld」に rtime フィールドと act メソッドを追加します。それぞれ private と public であることに注意してください。

2. 開発作業

準備:

シナリオ「GF-2-6」をコピーし、シナリオ名を「GF-2-7」に変更します。
Greenfoot でシナリオ「GF-2-7」を開きます。


コーディング:

「MyWorld」クラスに private int rtime フィールドを追加して下さい。
「MyWorld」クラスのコンストラクタで rtime フィールドに 1000 を代入して下さい。
「MyWorld」クラスに public void act メソッドを追加して下さい。
act メソッドで rtime フィールドをデクリメントして値を画面に表示して下さい。表示位置は左下なら任意の場所でOKです。
act メソッドに、rtime フィールドの値が 0 以下になったらタイムアップメッセージを出してプログラムを停止する処理を追加して下さい。メッセージ内容と表示位置は任意です。


テストとデバッグ:

Reset ボタンを押して初期化して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は Reset ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のクラス図とソースコード(MyWorld)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
