1. スプリント プランニング

詳細設計:

このスプリントでは、「APPLE」オブジェクトを動かします。

  • 「APPLE」クラスに public void act メソッドを追加して「APPLE」オブジェクトを動かします。動かし方は任意です。

スプリント バックログ:

  1. 「APPLE」クラスの act メソッドのコーディング( APPLE を動かす)

クラス図の作成:

  • 開発作業を行う前にクラス図を描いて下さい。
  • クラス図「GF-2-7」をコピーし、ファイル名を「GF-2-8」に変更します。
  • 「APPLE」に act メソッドを追加します。public であることに注意してください。

2. 開発作業

準備:

シナリオ「GF-2-7」をコピーし、シナリオ名を「GF-2-8」に変更します。
Greenfoot でシナリオ「GF-2-8」を開きます。


コーディング:

「APPLE」クラスに public void act メソッドを追加して下さい。
act メソッドで「APPLE」オブジェクトを動かして下さい。動きのアルゴリズムは任意とします。


テストとデバッグ:

Reset ボタンを押して初期化して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は Reset ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のクラス図とソースコード(APPLE)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
