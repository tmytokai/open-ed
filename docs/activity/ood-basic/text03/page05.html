1. スプリント プランニング

詳細設計:

このスプリントでは、「TARO」と「HANAKO」オブジェクトが「FRUIT」オブジェクトを食べる処理、つまり当たり判定を実装します。

  • 「HUMAN」クラスに private int fruits フィールドを追加します。fruits は自分が果物をいくつ食べたかを表す整数です。
  • 「HUMAN」オブジェクトが「FRUIT」オブジェクトに触れたらそのオブジェクトを消し、fruits フィールドをインクリメントする処理を書きます。

スプリント バックログ:

  1. 「HUMAN」クラスのコンストラクタのコーディング( fruits フィールドのリセット)
  2. 「HUMAN」クラスの act メソッドのコーディング(果物に触れたら果物消して fruits をインクリメント)

クラス図の作成:

  • 開発作業を行う前にクラス図を描いて下さい。
  • クラス図「GF-2-3」をコピーし、ファイル名を「GF-2-4」に変更します。
  • 「HUMAN」に fruits フィールドを追加します。private であることに注意してください。

2. 開発作業

準備:

シナリオ「GF-2-3」をコピーし、シナリオ名を「GF-2-4」に変更します。
Greenfoot でシナリオ「GF-2-4」を開きます。


コーディング:

「HUMAN」クラスに private int fruits フィールドを追加して下さい。
「HUMAN」にコンストラクタを追加して fruits をゼロクリアして下さい。
「HUMAN」の act メソッドの移動処理の下に、「FRUIT」型のオブジェクトに触れたら そのオブジェクトを削除し、fruitsをインクリメントする処理を書いて下さい。


テストとデバッグ:

Reset ボタンを押して初期化して下さい。
Run ボタンを１回押して動作確認して下さい。
上手く動作しなかった場合は Reset ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のクラス図とソースコード(HUMAN)を選んで下さい。
提出用にクラス図の画像ファイルをエクスポートして下さい。
指定場所に「クラス図」と「ソースコード」を貼り付けて下さい。
