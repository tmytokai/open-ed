オブジェクト指向言語の「オーバーロード(overload)」は日本語では「多重定義」と呼ばれていますが、一般的にはオーバーロードと呼ばれることが多いみたいですので、このアクティビティでもオーバーロードに呼び方を統一します。

なおオーバーロードの種類には主に「メソッドのオーバーロード」と「演算子のオーバーロード」がありますが、今回は「メソッドのオーバーロード」だけ取り扱います。
ただし「演算子のオーバーロード」の方も大変良く使われる機能ですので、もし今後必要になったら自分で調べて下さい。

さて「(メソッドの)オーバーロード」とは簡単に言うと

名前は同じだけど引数の個数や型が違うメソッドを複数個定義する機能

の事です。

では実際にオーバーロードの例を見てみましょう。以下の OLtest というクラスを定義します。

ソース1: 
public class OLtest {

    public void show(){
    	System.out.println("引数なし");
    }
    
    public void show( String a ){
    	System.out.println("引数は文字列クラス: " + a);
    }

    public void show( int a ){
    	System.out.println("引数は整数クラス: " +a);
    }

    public void show( int a, int b ){
    	System.out.println("引数は整数クラスx2: " +a + "-" + b);
    }
}

定義したメソッドの名前がすべて show であることに注意して下さい。

ではこの OLtest クラスを次の main メソッドから使ってみましょう。

ソース2: 
public static void main(String[] args) {

   OLtest ol = new OLtest();
   ol.show();
   ol.show( "ほげ" );
   ol.show( 123 );
   ol.show( 123, 456 );
}

ソース2の実行結果: 
引数なし
引数は文字列クラス: ほげ
引数は整数クラス: 123
引数は整数x2クラス: 123-456

このように、同じ名前のメソッド show を呼び出しているのに、引数の数や型が違うとそれらに対応した異なるメソッドが呼び出され、表示が変わります。

一方、オーバーロードを利用しない場合はソース 3 のようにメソッド名を変える必要があります。
もちろんこのソース 3 でも動きますが、ソース 4 のように引数の数や型毎にいちいち呼び出すメソッド名を変えないといけないのでプログラマが混乱する原因となりがちです。

ソース3: オーバーロードを使わない書き方 
public class OLtest {

    public void show1(){
    	System.out.println("引数なし");
    }
    
    public void show2( String a){
    	System.out.println("引数は文字列: " + a);
    }

    public void show3( int a){
    	System.out.println("引数は整数: " +a);
    }

    public void show4( int a, int b){
    	System.out.println("引数は整数x2: " +a + "-" + b);
    }
}

ソース4: 引数に合わせていちいちメソッド名を変える必要が出てくる 
public static void main(String[] args) {

   OLtest ol = new OLtest();
   ol.show1();
   ol.show2( "ほげ" );
   ol.show3( 123 );
   ol.show4( 123, 456 );
}
