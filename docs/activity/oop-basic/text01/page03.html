「インスタンス」とはクラス定義に含まれるフィールドの内容を記憶する領域のことで「オブジェクト」と同意義になります。

ただし、単純にそう言い切ってしまうことには注意が必要です。
何故かというと、クラス自体の事をオブジェクトと呼ぶ人もいるし、その両方を合わせてオブジェクトと呼ぶ人がいたりして、世界的な論争の元となっているからです。
あまりその論争に関わっていると話が進みませんので、とりあえず今回のアクティビティでは

インスタンス ＝ オブジェクト

と言い切ることにして、このページ以降は「オブジェクト」という用語は使わず「クラス」「インスタンス」という用語だけを使うことにします(※)。

※ 繰り返しますが クラス ＝ オブジェクト の意味で使ってる人もいますので技術書等を読む時は文脈に気を付けましょう。

では早速「new 演算子」を使って前ページで定義した Score クラスの「インスタンス」を作ってみましょう。

ソース1: 
public static void main(String[] args) {

	Score sc = new Score();  // (1)
	sc.math = 80; // (2)
	sc.english = 70; // (3)

	System.out.println( "math = " + sc.math+ " english = " + sc.english );
}

(1)の所でnew 演算子によって「 sc という Score クラスのインスタンス」がメモリ上に作られています。
これをC言語と比較すると

「クラス」→「構造体定義」

「new 演算子」 →「malloc関数」

「インスタンス」→「mallocでメモリ確保した構造体変数」

に相当します。

なおC言語では malloc したら free してメモリ解放する必要がありますが、Java の場合はインスタンスを使い終わったらメモリ領域がシステムにより自動的に開放されます(※)。

※ メモリ解放をプログラマが明示的にしないといけないかどうかは言語によります。
　例えば C++ の場合は delete を使って明示的なメモリ解放が必要です。

さて、一度インスタンスを作ってしまえば構造体変数の様にそのインスタンスのフィールドに対し

インスタンス名.フィールド名

でアクセス出来ます(ソース内の(2)、(3)の部分)。
