オブジェクト指向プログラミングとは


クラスを学ぶ前に、そもそも「オブジェクト指向プログラミング」とは何かについて考えてみましょう。

実は「オブジェクト指向プログラミング」には色々な解釈があって良く議論や争論のネタになるのですが、一般的には


お互いにデータやメッセージを交換しながら協調して動く複数のオブジェクトを上手く組み合わせて一つのソフトウェアを作りあげるプログラミング技法


がその説明になります。

それで肝心の「オブジェクト」とは何かですが、実は「オブジェクト」の定義も人によって違うのでこれも良く議論や争論のネタとなります。一般的には


世の中にある全てのものを表す概念


がその説明になります。例えば目で見て触れる事が出来るパソコンとかは当然オブジェクトですが、そのパソコンを作っているCPUとかマザーボードみたいなパーツもオブジェクトになります。
更に酸素・水素・炭素みたいな物質、歩く・座る・喜ぶみたいな行動、感情・意思・魂みたいな概念も全てオブジェクトです。

ただし概念的な事ばかり言ってたらいつまで経ってもプログラミング出来ませんので、このアクティビィティでは


「オブジェクト」は「クラス」と呼ばれるコードを元にして作られるメモり領域(＝インスタンス)


と具体的に言い切ってしまうことにします(嘘付くなと憤慨する人が出てくるかもしれませんが取り敢えず置いときます)。
「インスタンス」について詳しくは後のインスタンスのページで説明します。

それでこの「クラス」とはいくつかの変数群とそれらを操作する関数群をひとまとめにしたコードのことで、 C言語でいう構造体に関数を追加したものです(と言い切ってしまうとやっぱり怒る人がいます)。

例えば下がJava言語によるクラスの定義例です。
クラスの詳しい定義方法はこれから学んでいきますが、とりあえず int 型の math、english という変数から出来ている(C言語で言う)構造体の中に average() という double型の関数が追加されていることが分かります。 

クラスの定義例:

public class Score {
    public int math;
    public int english;
    
    public double average(){
    	return (math + english)/2.0;
    }
}

クラスを定義しておくと関連がある複数の変数や操作群をひとまとめに出来るのでソースコードがすっきりして綺麗になり、バグが減るというメリットがあります。
