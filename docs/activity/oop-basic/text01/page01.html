クラスを学ぶ前に、そもそも「オブジェクト指向プログラミング」とは何かについて考えてみましょう。

実は「オブジェクト指向プログラミング」には色々な解釈があって良く議論や争論のネタになるのですが、一般的には

「お互いにデータやメッセージを交換しながら協調して動く複数のオブジェクトを上手く組み合わせて一つのソフトウェアを作りあげるプログラミング技法」

がその説明になります。
それで肝心の「オブジェクト」とは何かですが、実は「オブジェクト」の定義も人によって違うのでこれも良く議論や争論のネタとなります。一般的には

「世の中にある全てのものを表す概念」

がその説明になります。
例えば目で見て触れる事が出来るパソコンとかは当然オブジェクトですが、そのパソコンを作っているCPUとかマザーボードみたいなパーツもオブジェクトになります。
更に酸素・水素・炭素みたいな物質、歩く・座る・喜ぶみたいな行動、感情・意思・魂みたいな概念も全てオブジェクトです。

ただし概念的な事ばかり言ってたらいつまで経ってもプログラミング出来ませんので、このアクティビィティでは

「オブジェクト」は「クラス」と呼ばれるコードを元にして作られるメモり領域

と具体的に言い切ってしまうことにします(※)。

※ 正確にはクラスを元にオブジェクトを作る方式の言語の事を「クラスベース」のオブジェクト指向プログラミング言語と呼びます。
　一方、「プロトタイプベース」というのもあります。
　Javascript がプロトタイプベース・オブジェクト指向プログラミング言語の代表例になります。

この「クラス」はいくつかの変数群とそれらを操作する関数群をひとまとめにしたコードのことで、例えば次は Java 言語によるクラスの定義例です。

Java によるクラスの定義例: 
public class Score {
    public int math;
    public int english;
    
    public double average(){
    	return (math + english)/2.0;
    }
}

「クラス」の詳しい定義方法はこれから学んでいきますが、コードの中に int 型の math、english という変数と average() という double型の関数が含まれていることが分かります。

この「クラス」を定義しておくと関連がある複数の変数や操作群をひとまとめに出来るのでソースコードがすっきりして綺麗になり、バグが減るというメリットがあります。
