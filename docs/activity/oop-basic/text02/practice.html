演習 2-2 (個人、30分): try の中を複数行にし、エラーが起きるとその時点で処理が中断されて catch にジャンプすることを確認しましょう。

新規のJavaプロジェクトを作ります。プロジェクト名は「OBSC-2-2」とします。
Main クラスを作成し、演習 2-1 の Main クラス を丸ごとコピーします。
try の hoge.txt をオープンしている行の「下」に以下の 2 行を追加します。 

FileReader in2 = new FileReader( "hoge2.txt" ); 
FileReader in3 = new FileReader( "hoge3.txt" );

ここまでで一旦実行確認します。
実行フォルダに空の hoge.txt ファイルを追加して実行し、エラーメッセージが変化したことを確認します。
同様に実行フォルダに空の hoge2.txt ファイルを追加して実行し、エラーメッセージが変化したことを確認します。
チーム内で答え合わせをし、提出用のソースコードを選んで下さい。
指定場所に「ソースコード」と「実行結果」を記入して下さい。


2. 標準例外

演習 2-3 (個人、30分): ゼロ除算例外を試してみましょう。

新規のJavaプロジェクトを作ります。プロジェクト名は「OBSC-2-3」とします。
Main クラスを作成し、演習 2-2 の Main クラス を丸ごとコピーします。
try の hoge.txt をオープンしている行の「上」に、以下の処理を追加します。

  • int 型変数 a にキーボードから数字を代入する
  • 1/a を画面に表示する

FileNotFoundException の catch 処理の下に ArithmeticException の catch 処理を追加します。
実行し、0 を入力したらゼロ除算例外、0以外を入力したら ファイル・ノットファウンド例外が起きることを確認して下さい。
チーム内で答え合わせをし、提出用のソースコードを選んで下さい。
指定場所に「ソースコード」と「実行結果(0を入力した時の表示)」を記入して下さい。


3. 例外の自作

演習 2-4 (個人、30分): 例外を自作してみましょう。

新規のJavaプロジェクトを作ります。プロジェクト名は「OBSC-2-4」とします。
Main クラスを作成し、演習 2-3 の Main クラス を丸ごとコピーします。
MyException クラスを作成してソース1をコピーします。
try の a に数字を入力している行の「下」に、以下の処理を追加します。

a が 3 だったら MyException 例外を投げる


ArithmeticException の catch 処理の下に MyException の catch 処理を追加します。
実行し、3 を入力したら自作例外、0 を入力したらゼロ除算例外、それ以外を入力したら ファイル・ノットファウンド例外が起きることを確認して下さい。
チーム内で答え合わせをし、提出用のソースコードを選んで下さい。
指定場所に「ソースコード」と「実行結果(3を入力した時の表示)」を記入して下さい。
