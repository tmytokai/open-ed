「量子化ビット数」 $q$ [bit] とは線形量子化した後のディジタルデータひとつ分を何 bit で記録するかを表す数字で、具体的には $f[i]$ の値域を均等に $2^q-1$ 分割することを意味します。

線形量子化ビット数 $q$ [bit] と対応する値域の分割数及び C言語における整数型の関係を示した表を以下に示します。
C 言語の場合は char 型(8 bit)、 short 型(16 bit)や int 型(32 bit)などの整数型の変数を使ってデータを記録します。

線形量子化ビット数 $q$ [bit] 値域の分割数 整数型変数
1 bit $2^1-1= 1$ 等分 (unsigned) char型
※ 8 bit 変数 x1 → 1 bitのデータ x 8
2 bit $2^2-1= 3$ 等分 (unsigned) char型
※ 8 bit 変数 x1 → 2 bitのデータ x 4
8 bit $2^8-1= 255$ 等分 (unsigned) char 型
16 bit $2^{16}-1= 65535$ 等分 (unsigned) short 型
24 bit $2^{24}-1= 16777215$ 等分 (unsigned) char型 x 3 個
※ 8 bit 変数 x 3 → 24 bitのデータ x 1
32 bit $2^{32} -1= 4294967295$ 等分 (unsigned) int 型

例えば前ページの図 1 の信号を $q$ = 2 bit で量子化した例を以下に示します。

前ページの図1を $q$ = 2 bit で量子化した例

$f[i]$ の値域を 0 から 1 までの範囲とし、それが $2^2-1 = 3$ 等分されるので、全てのディジタルデータは 0 か 1/3 か 2/3 か 1 のどれかの値に量子化される。
よって 2 bit あればディジタルデータひとつ分を記録する事が出来る。

例えば信号値 0 のデータを二進数 0b00、1/3 のデータを 0b01、2/3 のデータを 0b10、1 のデータを 0b11 で記録する事にすると、下の信号全体は二進数 0b001100011110 で表される。
