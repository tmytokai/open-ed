1. アクセス修飾子

演習 2-1 (個人、10分): 演習 1-4 で使った Score スーバークラスのフィールドのアクセス修飾子を private に変更するとどうなるか確認してみましょう。

プロジェクト「OELM-1-4」を丸ごとコピーし、名前を「OELM-2-1」とします。
Score スーパークラスの name, math, english のアクセス修飾子を private に変更します。
実行します。コンパイルエラーが出ますが、無視して実行してエラーメッセージを確認して下さい。
チーム内で答え合わせをし、提出用のソースコードを選んで下さい。
指定場所にと「エラーメッセージ」を記入して下さい。※ 「ソースコード」は必要ありません。

3. getter と setter

演習 2-2 (個人、15分): 「setter」を使って、Taro と Hanako サブクラスに表示されているエラーマークを消しましょう。

プロジェクト「OELM-2-1」を丸ごとコピーし、名前を「OELM-2-2」とします。
private なフィールドにはサブクラスからもアクセス出来ないので、name の値をセットする setter を Score クラスに追加します。
Taro クラスのコンストラクタ Taro() の中で setter を使って name フィールドに "太郎" を代入して下さい。
同様に Hanako クラスのコンストラクタ Hanako() の中で setter を使って name フィールドに "花子" を代入して下さい。
一旦実行します。Main クラスは変更していないので当然エラーが出ますが、Taro と Hanako のエラーマークが消えた事を確認して下さい。
チーム内で答え合わせをし、代表者は eclipse のスクリーンショットを保存して下さい。
指定場所に「ソースコード」「eclipse のスクリーンショット」を貼り付けて下さい。


演習 2-3 (個人、15分): 「getter」を使って全てのエラーマークを消しましょう。

プロジェクト「OELM-2-2」を丸ごとコピーし、名前を「OELM-2-3」とします。
name, math, english の値を取得する getter を Score クラスに追加します。
Main クラスの表示部で getter を使って各フィールド値を取得します。
実行します。
チーム内で答え合わせをし、提出用のソースコードを選んで下さい。
指定場所に「ソースコード」「実行結果」を貼り付けて下さい。
