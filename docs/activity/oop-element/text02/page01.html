今まで特に説明も無くフィールドとメソッドの前に「public」というキーワードをつけていましたが、この「public」は専門的には「アクセス修飾子」と呼ばれています。

この「アクセス修飾子」はフィールドやメソッドのアクセス権を決めるための修飾子で、大抵のオブジェクト指向言語には「public」「private」「protected」の3つの修飾子が用意されています。

さてアクセス修飾子のうち、public と private は次のような意味を持っています。

アクセス修飾子意味
publicフィールドやメソッドは他の全てのクラスやメソッドから自由にアクセスすることが出来る
privateフィールドやメソッドはそれらを定義したクラスの中からしかアクセス出来ない

なお「protected」は言語によって動作が微妙に異なるので今回のアクティビティでは詳しく説明しませんが、一般的に「protected」なフィールドやメソッドは他のクラスからは原則アクセス出来ないけどサブクラスからは例外的にアクセス出来るという「public」と「private」の中間の様な性質を持っています。

またクラスそのものにもアクセス修飾子を付けることが出来ますが(※)、これも言語によって動作が微妙に異なるので今回のアクティビティでは詳しく説明しません。興味のある人は自分で調べてみて下さい。

※ 今回のアクティビティでは常に public class 〜 でクラス定義しています。
