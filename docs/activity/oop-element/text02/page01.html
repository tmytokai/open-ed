アクセス修飾子


今まで特に説明も無くクラス名、フィールド、メソッドの前に「public」というキーワードをつけていましたが、この「public」は専門的には「アクセス修飾子」と呼ばれるものです。
参考までにクラスの定義方法を再掲します。 

クラスの基本的な定義方法:

アクセス修飾子 class クラス名{

    アクセス修飾子　型　フィールド名1;
    アクセス修飾子　型　フィールド名2;
    
    アクセス修飾子　戻り値の型　メソッド名1(引数){
        内容;
    }

    アクセス修飾子　戻り値の型　メソッド名2(引数){
        内容;
    }
}


この「アクセス修飾子」はそのクラスやフィールド、メソッドのアクセス権を決めるためのキーワードで、大抵のオブジェクト指向言語には「public」「private」「protected」の3つの修飾子が用意されています。

「public」なフィールドやメソッドは他の全てのクラスやメソッドから自由にアクセスすることが出来ます。

「private」なフィールドやメソッドはそれらを定義したクラスの中からしかアクセス出来ません。

「protected」は実は言語によって動作が微妙に異なるのですが、「protected」なフィールドやメソッドは他のクラスからは原則アクセス出来ないけどサブクラスからは例外的にアクセス出来る、という「public」と「private」の中間の様な性質を持っています。

なおクラスのアクセス権については少し複雑な話になるので今回のアクティビティでは説明を省いて「public class」のみを使用することにします。
興味のある人は自分で調べて下さい。
