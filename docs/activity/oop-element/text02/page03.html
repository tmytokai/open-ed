getter と setter

前のページで「フィールドは原則 private にして隠蔽しましょう」と書きましたが、それでもどうしても外部からフィールドにアクセスしたい場合があります。
その場合は「getter」と「setter」というテクニックを使うことが一般的です。

「getter」 (ゲッター）は private により隠蔽されたフィールドの値を外部から取得するために使うメソッドです。
例えばJavaではソース1の様に定義します。 

ソース1: getter の定義

public class HOGE{

  private int abc;

  // フィールド abc の getter
  // メソッド名は何でも良いが　get〜 とするのが一般的
  public int get_abc(){ return abc; }
}

それでHOGE以外のクラスや main メソッドなどから private なフィールド abc の値を取得したい場合はソース2のようにします。 

ソース2: getter の使い方

HOGE hoge = new HOGE();
int a = hoge.get_abc();

一方「setter」(セッター）は private により隠蔽されたフィールドの値を外部から変更するために使うメソッドです。
例えばJavaではソース3の様に書きます。 

ソース3: setter の定義

public class HOGE{

  private int abc;
  
  // フィールド abc の setter
  // メソッド名は何でも良いが set〜 とするのが一般的
  // 引数名は何でも良い。人によって名前付けの流儀もまちまち
  public void set_abc( int _abc ){ abc = _abc; }
}

それでHOGE以外のクラスや main メソッドなどから private なフィールド abc の値をセットしたい場合はソース4のようにします。 

ソース4: setter の使い方

HOGE hoge = new HOGE();
hoge.set_abc( 123 );

さて実はこの getter/setter のテクニックは最近は世界中のプログラマーから嫌われていて、C# や swift などの新し目の言語では代わりに「プロパティ」という似たような機能が用意されています(※)。
例えば C# のプロパティはソース5の様に書きます。

※ 古めのC++やJavaには有りません

詳しくはここでは取り扱いませんが、「プロパティ」は最近のプログラミングでは必須の機能ですので、詳細については自分で調べておきましょう。 
ソース5: C#のプロパティの例

定義:

class HOGE{

  private int abc;

  public int Abc{
      get { return abc; }
      set { abc = value; }
  }
}

使い方:

HOGE hoge = new HOGE();
int a = hoge.Abc;
hoge.Abc = 123;
