  2. 学習項目: [2] カプセル化

アクセス修飾子には「public」「private」(と「protected」)がありますが、不用意にフィールドやメソッドに対して「public」を付けると、他のクラスから好き勝手にアクセス出来るようになるため誤動作やバグの原因になります。

さらにデバッガを使わない限り、一体どこからアクセスされたのか分からないのでデバッグ作業も困難になります。

従って

  1. フィールドやメソッドのアクセス権は原則として「private」にする
  2. 「public」の使用は必要最低限に留める

ことで他のクラスから簡単にアクセス出来ないようにすることがオブジェクト指向の世界では推奨されています。

このように、アクセス修飾子をうまく使ってクラス内でしている事を隠蔽し、外から中身を見えなくすることを「カプセル化」といいます。

カプセル化されたクラスは「public」を付けたメソッドやフィールド経由でしか外からアクセス出来なくなるので、クラス間の独立性が上がってデバッグ作業が楽になるメリットがあります。

同様に、あるクラスに行なった変更が他のクラスに及ぼす影響も小さくなるので、多人数での開発がやりやすくなるメリットもあります。
