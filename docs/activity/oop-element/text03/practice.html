5. インヘリタンスによるポリモーフィズム

演習 3-1 (個人、15分): インヘリタンスによるポリモーフィズムの便利さを確認してみましょう。

プロジェクト「OELM-2-3」を丸ごとコピーし、名前を「OELM-3-1」とします。
Main クラスの main メソッドの上に public static void show( Score s ) メソッドを追加します。
main メソッド内の taro の情報表示部を show メソッド内に移動し、ページ5のソース 5を参考にして Score クラスのインスタンス s の情報表示に変えて下さい。
main メソッド内の hanako の情報表示部を消して下さい。
main メソッド内から show メソッドを呼び出して、taro と hanako の情報を表示して下さい。
実行し、便利さ確認して下さい。
チーム内で答え合わせをし、提出用のソースコードを選んで下さい。
指定場所に「Main クラスのソースコード」を記入して下さい。 ※ 他のクラスのソースは必要ありません。


演習 3-2 (個人、15分): サブクラスを 3 つに増やし、インヘリタンスによるポリモーフィズムとインスタンス配列を組み合わせると便利な事を確認してみましょう。

プロジェクト「OELM-3-1」を丸ごとコピーし、名前を「OELM-3-2」とします。
Taro クラスを丸ごとコピーして Jiro クラスを作ります。name には "次郎" をセットして下さい。
ページ5のソース 6 を参考にし、長さ 3 の Score クラスのインタンス配列 score[3] を定義して下さい。
同様にソース 6 を参考にし、score[0] に Taro クラス、score[1] に Hanako クラス、 score[2] に Jiro クラスのインスタンスを new して代入して下さい。
同様にソース 6 を参考にし、for ループ内で show メソッドを呼び出し、各score[i] の情報を表示して下さい。
実行し、便利さを確認して下さい。
チーム内で答え合わせをし、提出用のソースコードを選んで下さい。
指定場所に「ソースコード」と「実行結果」を記入して下さい。


6. オーバーライドによるポリモーフィズム

演習 3-3 (個人、10分): オーバーライドを使って、次郎の合計点が本来の2倍になるようにしてみましょう。

プロジェクト「OELM-3-2」を丸ごとコピーし、名前を「OELM-3-3」とします。
Score クラスの total メソッドをJiro クラスの中でオーバーライドし、合計点の 2 倍を retrun して下さい。(ヒント) getter を使う必要有
実行し、太郎と花子は従来通りだけど次郎の合計点だけが本来の2倍になっていることを確認して下さい。
チーム内で答え合わせをし、提出用のソースコードを選んで下さい。
指定場所に「ソースコード」と「実行結果」を記入して下さい。
