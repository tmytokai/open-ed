フィールドのポリモーフィズム

まず「同じ名前のフィールドによるポリモーフィズム」について例を示します。

これは「同じ名前のフィールドを使ってもインスタンス毎に動作が異なる」という性質のことです。
例えば次のmainメソッドを見て下さい(ソース1)。 
ソース1:

public static void main(String[] args) {

        FRUIT apple = new FRUIT();
        FRUIT orange = new FRUIT();

        apple.name = "りんご";
        orange.name = "みかん";

        System.out.println(apple.name + "美味しい");
        System.out.println(orange.name + "美味しい");
    }

apple と orange はどちらも FRUITクラス(ソース2)のインスタンスで、同じ name フィールドを表示しています。 ただし name の値は異なります。 

ソース2:

public class FRUIT {

    public String name;
}

実行結果は次の通りです。 

結果1:

りんご美味しい
みかん美味しい

この様に同じ name フィールドを表示しているのにインスタンス毎に動作が異なっています。
これが「同じ名前のフィールドによるポリモーフィズム」です。

ただし今回の例はわざわざクラスを使わなくてもC言語でいう構造体だけで十分実現可能な話なので、「同じ名前のフィールドによるポリモーフィズム」は一般的にはポリモーフィズムとしてみなされていない様です。
