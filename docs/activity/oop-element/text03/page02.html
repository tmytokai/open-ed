まず単純な例として「フィールドによるポリモーフィズム」を示します。

これは「同じクラスの同じ名前のフィールドを使ってもインスタンス毎に動作が異なる」という性質のことで、実は今まで既に演習で体験済みです。

例えばソース 1 の FRUIT クラスに対してソース 2 の main メソッドを実行してみます。

ソース 1 : 
public class FRUIT {

    public String name;
}

ソース 2 : 
public class Main{

	public static void main(String[] args) {

	        FRUIT apple = new FRUIT();
	        FRUIT orange = new FRUIT();

	        apple.name = "りんご";
	        orange.name = "みかん";

	        System.out.println( apple.name + "美味しい");
	        System.out.println( orange.name + "美味しい");
	}
}

当然実行結果は次の様になります。

実行結果 1 : 
りんご美味しい
みかん美味しい

この様に、同じ FRUIT クラスの同じ name フィールドを表示しているのにインスタンス apple と orange で表示内容が異なっています。

ただし普通はカプセル化によりフィールドに直接アクセスすることは無いので、「フィールドによるポリモーフィズム」はポリモーフィズムの例として挙げられることはあまり無いようです。
