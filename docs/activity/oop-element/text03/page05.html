「オーバーライド」とはスーパークラスのメソッドをサブクラスで上書きし、そのサブクラスのインスタンスを呼び出した時に書き換えたサブクラスの方のメソッドを呼び出す機能で、これもポリモーフィズムのひとつです。
以前勉強したオーバロードと言葉は似ていますが全く違う機能です。

今まで学んできた「メソッドによるポリモーフィズム」と「インヘリタンスによるポリモーフィズム」においてメソッドを呼び出した時に動作が異なるのは、結局の所フィールド値が異なることがその理由でしたが、「オーバーライド」を使うとメソッドの動きを根本的に変えることが可能です。

では具体的な例を使って「オーバライド」について説明していきますが、前のページで使った FRUIT、APPLE、ORANGE クラスはそのまま使うことにします。
また main メソッドもソース 4 または 5 をそのまま使います。

参考までに実行結果をもう一度示します。

実行結果 1 : 
りんご美味しい
みかん美味しい

さて、ここで ORANGE サブクラスのインスタンスを eat() したら不味かったという問題が生じましたので、 この時だけ「みかん不味い」と表示したいと思います。

もしオーバーライドの事を知らないプログラマーなら FRUIT スーパークラスの eat() メソッドを多分次のソース 1 の様に書き換えるでしょう。

ソース 1: 非常に良くない例 
public class FRUIT {

    public String name;

    public void eat(){

        if( name == "みかん" ) System.out.println(name+"不味い");
        else System.out.println(name+"美味い");
    }
}

しかしこのコードには大問題があります。というのも

  • 更にもし「いちご」サブクラスを追加した時に「いちごまあまあ」と表示したいとしたら eat() に別の if 文を追加する必要がある
  • 更にもし「バナナ」サブクラスを追加した時に「バナナ腐ってる」と表示したいとしたら eat() に別の if 文を追加する必要がある
  • 更にもし・・・

というように、サブクラスを追加する度に if 文が eat() に追加されて行くことになり、ソースの見た目や独立性、保守性が最悪なコードになるからです。

一般的にスーパークラスのメソッド内ではサブクラス別の処理を書いてはいけません。

そこで、ある一部のサブクラスのメソッドだけスーパークラスのメソッドと違う動作をさせたい時は、スーパークラスのメソッドをサブクラスで「オーバーライド」、つまり上書きします。

ソース 2 がスーパークラスの eat() を ORANGE サブクラスの eat() でオーバーライドした例です。

ソース 2 : スーパークラスの eat() を ORANGE サブクラスの eat() でオーバーライド 
public class ORANGE extends FRUIT {

    public ORANGE(){
        name = "みかん";
    }

    public void eat(){
        System.out.println(name+"不味い");
    }
}

すると ORANGE サブクラスのインスタンスを eat() した時はスーパークラスの eat() では無く、代わりに ORANGE サブクラスの eat() が呼び出され次のような表示結果になります。

実行結果 2 : 
りんご美味しい
みかん不味い
