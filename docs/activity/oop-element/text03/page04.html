オブジェクト指向言語では「サブクラスのインスタンスをスーパークラスのインスタンスと見なして作る」ことが出来ます。
この機能を利用すると、前のページと同様に「同じ名前のメソッドを使ってもインスタンス毎に動作を異ならせることが可能」になります。

といきなり言っても何のことか良く分からないと思うので、これから具体的なコードを使って段階的に説明していきます。

まず大元の FRUIT スーパークラスをソース 1 で定義します。

ソース 1 : FRUIT スーパークラス 
public class FRUIT {

    public String name;

    public void eat(){
        System.out.println(name+"美味い");
    }
}

この FRUIT スーパークラスを継承させて APPLE サブクラス(ソース 2 )と ORANGE サブクラス(ソース 3 )を作ります。

ソース 2 : APPLE サブクラス 
public class APPLE extends FRUIT{

    public APPLE(){
        name = "りんご";
    }
}

ソース 3 : ORANGE サブクラス 
public class ORANGE extends FRUIT{

    public ORANGE(){
        name = "みかん";
    }
}

次に main メソッドをソース 4 に示しますが、今まで使ってきた main メソッドと違いapple と orange が FRUIT スーパークラスのインスタンスになっていることに注目して下さい。

ソース 4 : main メソッド 
public static void main(String[] args) {

	FRUIT apple = new APPLE();   // ※ 今までは「APPLE apple = new APPLE();」 だった！
	FRUIT orange = new ORANGE(); // ※ 今までは「ORANGE orange = new ORANGE();」 だった！

	apple.eat();
	orange.eat();
}

すると「apple と orange は見かけ上は FRUIT スーパークラスのインスタンスだけど、実態はそれぞれ APPLE、ORANGE サブクラスのインスタンス」なので、同じ eat() メソッドを呼び出しているのに apple と orange で結果が変わります(実行結果 1 )。

実行結果 1 : 
りんご美味しい
みかん美味しい

なお、インヘリタンスによるポリモーフィズムを応用すると、次のソース 5 の様にスーパークラスのインスタンス配列と for ループを使って異なるサブクラスの複数のインスタンスを一元管理出来るようになります。

このテクニックは非常に良く使われますので覚えておくと良いでしょう。

ソース 5 : main メソッド 改 
public static void main(String[] args) {
	        
        FRUIT fruit[] = new FRUIT[2];

        fruit[0] = new APPLE();
        fruit[1] = new ORANGE();
        
        for(int i=0;i < 2;++i) fruit[i].eat();
}

※1 オブジェクト指向言語ではこういう状況では普通は配列 + for ループではなくて vector クラス + iterator クラスを使います。詳細は自分で調べて下さい。

※2 前のページのソース 3 と似ていますが、ソース 3 では FRUIT クラスのインスタンスを FRUIT の配列に代入していたのに対し、ここでは APPLE と ORANGE のインスタンスを FRUIT の配列に代入しているという違いが有ります。この違いを良く覚えておきましょう。
