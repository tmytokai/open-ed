  2. 学習項目: [3] ポリモーフィズム

次に「同じ名前のメソッドによるポリモーフィズム」について簡単な例を示します。
これは「同じ名前のメソッドを使ってもインスタンス毎に動作が異なる」という性質のことで、やはり今まで既に演習で体験済みです。

例えばソース 1 の FRUIT クラスに対してソース 2 の main メソッドを実行してみます。

ソース 1 : 
public class FRUIT {

    public String name;

    public void eat(){
        System.out.println( name + "美味しい");
    }
}

ソース 2 : 
public static void main(String[] args) {

        FRUIT apple = new FRUIT();
        FRUIT orange = new FRUIT();

        apple.name = "りんご";
        orange.name = "みかん";

        apple.eat();
        orange.eat();
}

当然実行結果は次の様になります。

実行結果 1 : 
りんご美味しい
みかん美味しい

この様に同じ eat() メソッドを呼び出しているのに、インスタンス apple と orange で結果が異なっています。
これが「同じ名前のメソッドによるポリモーフィズム」です。

ちなみに、次のソース 3 の様にインスタンス配列と for ループを使って main メソッドを書き換えることも可能です。

ソース 3 : main メソッド 改 
public static void main(String[] args) {

        FRUIT fruit[] = new FRUIT[2];

        fruit[0] = new FRUIT();
        fruit[1] = new FRUIT();

        fruit[0].name = "りんご";
        fruit[1].name = "みかん";

        for(int i=0;i < 2;++i) fruit[i].eat();
}

※ オブジェクト指向言語ではこういう状況では普通は配列 + for ループではなくて vector クラス + iterator クラスを使います。詳細は自分で調べて下さい。
