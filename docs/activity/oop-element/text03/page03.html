メソッドのポリモーフィズム

次に「同じ名前のメソッドによるポリモーフィズム」について例を示します。

これは「同じ名前のメソッドを使ってもインスタンス毎に動作が異なる」という性質のことです。
例えば次のmainメソッドを見て下さい(ソース1)。 

ソース1:

public static void main(String[] args) {
	        
        FRUIT fruit[] = new FRUIT[2];
        fruit[0] = new FRUIT( "りんご" );
        fruit[1] = new FRUIT( "みかん" );
        
        for(int i=0;i<2;++i) fruit[i].eat();
}

この fruit[0] と fruit[1] は同じ FRUIT クラス(ソース2)のインスタンス配列で、同じ eat メソッドが呼び出されています。
ただしコンストラクタで違う名前をセットしています。 

ソース2:

public class FRUIT {

    public String name;


    public FRUIT( String n ){
        name = n;
    }

    public void eat(){
        System.out.println( name + "美味しい");
    }
}

実行結果は次の通りです。 

結果1:

りんご美味しい
みかん美味しい
