1. スプリント プランニング

詳細設計:

このスプリントでは、「BALL」オブジェクトを作成して Prefab化します。

  • 「BALL」の画像は任意とします。
  • 「BALL」は常に自分から見て上方向に動き続けます。
  • 「BALL」は他のオブジェクトをすり抜けます。
  • 「BALL」は一定時間後に自動的に削除されます。
  • 移動速度(speed) は任意とします。

スプリント バックログ:

  1. 「BALL」オブジェクトを作成して Prefab 化

2. 開発作業

準備:

プロジェクト「UNITY-3」に対して作業を続けます。


スプリントバックログ 1: 「BALL」オブジェクトを作成して Prefab 化

画像を用意し「BALL」に名前変更して下さい(※ 拡張子は変更しないよう注意)。
画像を「BALL」オブジェクト化して下さい。サイズは適宜変更して下さい。位置は適当で結構です。
「BALL」オブジェクトに Rigidbody 2D をアタッチして重力(Gravity Scale)を 0 にして下さい。なおConstraints → Freeze Rotation → Z にチェックは入れないで下さい。
「BALL」オブジェクトに Circle Collider 2D をアタッチして下さい。なおis Trigger をチェックして下さい。衝突範囲は適宜調整して下さい。
「BALL」スクリプトを新規作成して 「BALL」オブジェクトにアタッチして下さい。
「BALL」スクリプトを開き、Start メソッドで次のように上方向に velocity をセットします。speed の値は任意です。

float speed = 任意;
GetComponent<Rigidbody2D>().velocity = transform.up * speed; // transform.up でオブジェクトの上の方角が取得できる

このままだと「BALL」が発射されてから永久に動き続けるので、上のコードのすぐ後ろに「数秒したら自動で自分自身を削除する」プログラムを書いて下さい。
この時点で一度実行し、「BALL」が上に動いて設定した秒数後に Hierarchy ウィンドウから「BALL」の表示が消える事を確認して下さい。
実行を停止し、「BALL」を Prefab 化し、オブジェクトを削除して下さい。


テストとデバッグ:

実行ボタンを押して動作確認して下さい。
上手く動作しなかった場合は実行ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のスクリプト(BALL.cs)を選んで下さい。
指定場所に「ソースコード」を貼り付けて下さい。
