1. スプリント プランニング

詳細設計:

このスプリントでは、「TARO」が「BALL」を投げます。

  • 「BALL」は「TARO」が向いている方向に投げます。
  • 「BALL」を投げる時間間隔は任意です。

スプリント バックログ:

  1. 「BALL」のインスタンス化

2. 開発作業

準備:

プロジェクト「UNITY-3」に対して作業を続けます。


スプリントバックログ 1: 「BALL」のインスタンス化

「TARO」スクリプトをエディタで開いてください。
「public GameObject ball;」 フィールドを追加して下さい。
「BALL」を投げる時間間隔(秒数)を表す「private float nextball;」 フィールドも追加し、Start メソッドで 0 クリアして下さい。
FixedUpdate メソッドの左右の向きを変える処理の後ろに次のプログラムを書いて下さい。
nextball 秒の間「BALL」を投げれなくするための処理です。
間隔の秒数は任意で結構です。 

nextball -= Time.fixedDeltaTime;
if (nextball <= 0 && Input.GetButton ("Fire1") ){
   nextball = 次にBALLを投げるまでの間隔(秒);

}

上で書いたカッコの中の続きに「TARO が向いている方向に BALL をインスタンス化して投げる」プログラムを書いて下さい。

なお、「BALL」を表示する座標は「TARO」と同じ座標にしたいので transform.position で取得できます。
「TARO」が向いている方向は GetComponent<SpriteRenderer>().flipX が true か false かで分かります。
回転角度は左向きなら Quaternion.Euler(0f,0f,-90f) 、右向きなら Quaternion.Euler(0f,0f,90f) で取得できます。

スクリプトを保存して Unity エディタに戻って下さい。
「TARO」の Inspector ウィンドウを開き、TARO(Script) の Ball に「BALL」Prefab を DnD して下さい。


テストとデバッグ:

実行ボタンを押して動作確認して下さい。
上手く動作しなかった場合は実行ボタンを押して動作を止めてデバッグして下さい。


3. リリース

チーム内で答え合わせをし、提出用のスクリプト(TARO.cs)を選んで下さい。
指定場所に「ソースコード」を貼り付けて下さい。
