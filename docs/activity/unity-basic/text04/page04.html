1. スプリント プランニング

詳細設計:

このスプリントでは、「TARO」オブジェクトを作って左右に動かします。

  • TARO の画像は任意とします。
  • 重力は 1 とします(デフォルト値のまま)。
  • 移動速度(speed) は任意とします。

また TARO が左右から画面外に出られないように左右方向に見えない壁オブジェクトを作って置き、TARO を画面内に閉じ込めます。

  • 壁オブジェクトの名前は「left」「right」とします。

スプリント バックログ:

  1. 「TARO」オブジェクトの作成
  2. 左右方向の壁オブジェクトの作成

2. 開発作業

準備:

プロジェクト「UNITY-3」に対して作業を続けます。


スプリントバックログ 1: 「TARO」オブジェクトの作成

画像を用意し「TARO」に名前変更して下さい(※ 拡張子は変更しないよう注意)。
画像を「TARO」オブジェクト化して下さい。サイズは適宜変更して下さい。位置は適当で結構です。
「TARO」オブジェクトに Rigidbody 2D をアタッチして重力(Gravity Scale)が 1 なのを確認し、Constraints → Freeze Rotation → Z にチェックを入れて下さい。
「TARO」オブジェクトに Box Collider 2D をアタッチして下さい。衝突範囲は適宜調整して下さい。
「TARO」スクリプトを新規作成し、「TARO」オブジェクトにアタッチして下さい。
「TARO」スクリプトをエディタで開き、 void Update() の下に void FixedUpdate() メソッドを追加し、FixedUpdate メソッド内に「方向ボタンで左右に動かす」プログラムを記述して下さい。 speed は任意です。
ただし今回は重力があるため y 軸方向の速度は物理演算エンジンが自動計算してくれます。
従って次のようなコードを書いて y 軸方向の速度は変更しないようにして下さい。 

float speed = 任意;
float x = Input.GetAxisRaw ("Horizontal");  // x 方向は今までやってきた様に GetAxisRaw で取得
float y = GetComponent<Rigidbody2D>().velocity.y; // y 方向は物理演算エンジンに任せる
GetComponent<Rigidbody2D> ().velocity = new Vector2 (x*speed, y);

その続きで、「左ボタンを押していたら左を向く、右ボタンを押していたら右を向く」プログラムを記述して下さい。


スプリントバックログ 2 : 左右方向の壁オブジェクトの作成

まず左から作ります。Hierarchy ウィンドウで右クリック → Create Empty で空のオブジェクトを作り、「left」に名前を変更して下さい。
「left」オブジェクトに Box Collider 2D をアタッチし、衝突範囲を表示画面の縦方向いっぱいに引き伸ばしてください。
前ページと同じ理由で摩擦があると TARO が張り付くので、前ページで作った「SLIP」マテリアルを Box Collider 2D の Material に DnD して下さい。
「left」オブジェクトを画面外左に移動して下さい。
コピーするなりして「right」オブジェクトも作成して下さい。


テストとデバッグ:

実行ボタンを押して動作確認して下さい。
上手く動作しなかった場合は実行ボタンを押して動作を止めてデバッグして下さい。
