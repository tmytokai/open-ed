1. 準備

まず準備です。
演算テスト用に別の Script Graph を用意します。

アセットフォルダの空いている部分を右クリックしてメニューを表示し、Create → Visual Scripting → Script Graph を選ぶ
アセットフォルダ内に「New Script Graph」というアセットが出来るので「Math Script Graph」に名前を変更する
ヒエラルキーの PROGRAM をクリックしてインスペクタを表示する
Script Machine の Graph の Variable Script Graph と書いてあるセルにアセットフォルダの Math Script Graph をドロップする
すると Math Script Graph が実行されるようになる

4. 乱数の生成

乱数を生成するには Random:Range ユニット(random:rangeで検索)を使います。
ただし Max Inclusive 版と Max Exclusive 版の2通りがあるので、使い分けについて説明します。

まず Max Inclusive 版が出力する乱数は小数(Float型)です。
一方 Max Exclusive 版が出力する乱数は整数(Integer型)です。

それでパラメータの Min にはどちらの版も最小値を入れます。
一方、Max には最大値を入れます。
ただし Max Inclusive 版はMAX 以下の小数、 Max Exclusive 版はMAX よりも小さい整数がランダムに出てきます。

例えば図3のスクリプトを実行して下さい。
実行する度に違う数字が表示されますが、Max Inclusive 版は 0 から 2 までの小数、Max Exclusive 版は 0 か 1 のどちらかの整数が出力されます。


図3. 乱数の生成

ところで Random:Range ユニットにコントロール入出力ポートが付いていることに注目してください。

Random:Range ユニットは実行する度に異なる乱数を生成します。
従って図4の様にフローに Random:Range ユニットを組み込まない時は Debug:Log を実行する度に Random:Range ユニット が実行されて違う数字が表示されます。
一方、図5の様に Random:Range ユニットをフローの中に組み込むと 1 回しか呼び出されないので、最初に生成した数字がずっと保持されて同じ数字が表示されます。
