1. スプリント プランニング

詳細設計:

このスプリントでは、「APPLE」オブジェクトを動かします。

  • 「APPLE」は常に自分から見て上方向に動き続けます。
  • 「APPLE」は壁や ORANGE とぶつかると跳ね返ります。
  • 「APPLE」にセットするマテリアルの名前は「BOUND」とします。Friction(摩擦)は 0 、Bounciness(弾性)は 1 にします。
  • 「ORANGE」は最初の場所に静止したままです。

スプリント バックログ:

  1. 「APPLE」が跳ね返るように動かす
  2. 「ORANGE」を最初の位置に固定する

2. 開発作業

準備:

プロジェクト「UNITY-2」に対して作業を続けます。


スプリントバックログ 1: 「APPLE」が跳ね返るように動かす

「APPLE」スクリプトを新規作成して 「APPLE」 Prefab にアタッチして下さい。
「APPLE」スクリプトを開き、Start メソッドで次のように上方向に velocity をセットします。speed の範囲はランダムです。

float speed = Random.Range( 0f, 最大速度 );
GetComponent<Rigidbody2D>().velocity = transform.up * speed; // transform.up でオブジェクトの上の方角が取得できる

「BOUND」マテリアルを新規作成して下さい( Physics Material ではなくて Physics Material 2D にするのを間違わないこと)。
「BOUND」マテリアルをクリックし、 Friction(摩擦)を 0 、Bounciness(弾性)を 1 にします。
「APPLE」Prefabをクリックし、 Inspector ウィンドウの Circle Collider 2D の Material に「BOUND」を DnD します。
(実行した際に跳ね返り方が弱い場合) Unity エディタの Edit メニュー → Project Settings → Physics2D の Velocity Threshold の値を 0.0001 に変更して下さい。


スプリントバックログ 2 : 「ORANGE」を最初の位置に固定する

「ORANGE」Prefabをクリックし、 Rigidbody 2D の Constraints → Freeze Rotation の x と y にチェックを入れます。


テストとデバッグ:

実行ボタンを押して動作確認して下さい。
上手く動作しなかった場合は実行ボタンを押して動作を止めてデバッグして下さい。
