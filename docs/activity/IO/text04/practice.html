2. 出力


演習 4-1 (個人PC、20分): Arduino UNO から Bluetooth 経由でタブレットやスマートフォンなどのモバイル端末に文字列を送ってみましょう。

仕様は以下の通りです。

仕様:

(1) Bluetooth 機器として「ＲＮ－４２使用Ｂｌｕｅｔｏｏｔｈ無線モジュール評価キット」を使用する。

(2) プロファイルは SPP を使用する。

(3) モバイル端末はファシリテーターが用意する。

(4) 100ミリ秒毎に Arduino UNO 側からモバイル端末側へ数字がカウントアップしてが送られる( 0, 1, 2, ・・・みたいに)。

データシートや Web 等で回路図を調べて回路を組んで下さい。
回路を組んだらファシリテータの確認を受けて下さい。
プログラムを書いて Arduino UNO に転送して下さい。

　(注意) Arduino UNO 側の RX ピンのジャンパケーブルを外さないとエラーが出ます。

モバイル端末との通信を確認して下さい。
動作したらファシリテータの確認を受けて下さい。
ソースコードをドキュメントにコピーして下さい。
回路の全体写真を撮ってドキュメントファイルに貼りつけて下さい。
ドキュメントファイルに「回路の全体写真」と「ソースコード」が記入されていることを確認して下さい。


3. 入力


演習 4-2 (個人PC、20分): Arduino UNO を Bluetooth 経由でモバイル端末から制御してみましょう。

仕様は以下の通りです。

仕様:

(1) 回路は演習4-1とほぼ同じだが Arduino UNO の適当な GPIO に LED をつなげる。

(2) プロファイルは SPP を使用する。

(3) モバイル端末から文字 'a' が送られたら LED を点灯する。

(4) モバイル端末から文字 'b' が送られたら LED を消灯する。

仕様に従って回路を修正して下さい。
回路を組んだらファシリテータの確認を受けて下さい。
プログラムを書いて Arduino UNO に転送して下さい。

　(注意) Arduino UNO 側の RX ピンのジャンパケーブルを外さないとエラーが出ます。

モバイル端末との通信を確認して下さい。
動作したらファシリテータの確認を受けて下さい。
ソースコードをドキュメントにコピーして下さい。
回路の全体写真を撮ってドキュメントファイルに貼りつけて下さい。
ドキュメントファイルに「回路の全体写真」と「ソースコード」が記入されていることを確認して下さい。
