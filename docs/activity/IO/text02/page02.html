2. 出力

前述した様に RS-232C では Rx/Tx ピンを使って情報を送受信します。
ここで Rx/Tx ピンの正式名称と用途は以下の通りです。

Rx/Tx ピンの正式名称と用途:

Rx : Recieved eXchange ・・・受信に使う

Tx : Transmit eXchange ・・・送信に使う

さて、ある機器やマイコン、IC等から別の機器やマイコン、IC等に情報を送りたい場合は、送信側の Tx ピンと受信側の Rx ピンを線1本で繋ぎ、メーカーが用意したライブラリの関数を使って情報を送るだけです。

関数の書式は大抵は次のようになっています。

シリアル通信で情報を送る:

関数名( ポート番号(※)、変数や配列や文字列 )

※マイコンボードやICの場合はシリアルポートがひとつしか無いことが多いのでポート番号は省略されることもある

ただし GPIO の時と同様に電圧レベルに注意する必要があります。

PC等の比較的大きい機器で使われている RS-232C の電圧レベルは -15V 〜 +15V なのですが、マイコンやIC等の Rx/Tx ピンの電圧レベルは一般に 0V 〜 5V 又は 0V 〜 3.3V になります。

従ってRS-232Cの電圧レベルが異なる場合は、機器や回路の間に電圧レベルの変換ICを入れないと通信出来ないばかりか、最悪機器や回路が壊れるので注意して下さい。
