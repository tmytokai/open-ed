アクティビティ: 入出力インターフェース
学習項目: [2] RS-232C
演習時間: 70 分

演習


1. RS-232Cとは

演習 1 (個人PC、10分): 「RS-232C 以外のシリアルインターフェース」を最低ひとつ調べてドキュメントファイルに記述してください。


2.出力

演習 2 (個人PC、40分): Arduino UNO の Tx ピンと PC 側の Rx ピンを繋いで Arduino UNO から PC へ文字列を送ってみましょう。

仕様:

(1) 電圧レベル変換ICとして ADM3202AN を使う

(2) T1in と T1out を使う

(3) PCにはシリアルポートが無いのでUSB-シリアル変換ケーブルを使う

(4) PC 側の通信アプリとして teraterm を使う

(5) 100ミリ秒毎に Arduino UNO 側から PC 側へ適当な文字列が送られる

まずホワイトボードに回路の設計図を描いて下さい。
チームメンバー全員が描いたらファシリテーターの確認を受け、自分が描いた設計図の写真を撮って保存して下さい。

次に設計図を見ながら回路を組んで下さい(注意1、2)。
チームメンバー全員が組んだらファシリテーターの確認を受け、自分が組んだ回路の全体写真を撮って保存して下さい。

(注1) データシートでは3.3V駆動になっていますが 5V駆動にして下さい。
(注2) ジャンパーケーブルが多いと見づらいので、1ピン(C1+) - 3ピン(C1-)、4ピン(C2+) - 5ピン(C2-)、16ピン(Vcc) - 2ピン(V+) はコンデンサで直結して下さい(ショートさせない様に気を付ける)。

次にプログラムを書いて転送して下さい(注3)。
チームメンバー全員が動作したら各自ファシリテーターの動作確認を受けて下さい。

(注3) Arduino UNO 側のRX/TXピンのジャンパケーブルを外さないとエラーが出るので注意。

最後にドキュメントファイルに「設計図の写真」と「回路の写真」と「ソースコード」をコピーして保存してください。


3.入力

演習 3 (個人PC、20分): Arduino UNO の Rx ピンと PC 側の Tx ピンを繋いで PC から Arduino UNO を制御してみましょう。

仕様:

(1) 回路は演習2とほぼ同じだが、今回は R1in と R1out を使う(T1in と T1outからケーブルを付け替える)

(2) Arduino UNO の適当な GPIO に LED をつなげる

(3) PC から 'a' が送られたら LED を点灯する

(4) PC から 'b' が送られたら LED を消灯する

まず仕様に従って回路を修正して下さい。
チームメンバー全員が組んだらファシリテーターの確認を受け、自分が組んだ回路の全体写真を撮って保存して下さい。

次にプログラムを書いて転送して下さい(注1)。
チームメンバー全員が動作したら各自ファシリテーターの動作確認を受けて下さい。

(注1) Arduino UNO 側の RX/TXピンのジャンパケーブルを外さないとエラーが出るので注意。

最後にドキュメントファイルに「設計図の写真」と「回路の写真」と「ソースコード」をコピーして保存してください。
