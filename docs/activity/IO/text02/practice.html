2. 出力


演習 2-1 (個人PC、30分): Arduino UNO の Tx ピンと PC 側の Rx ピンを繋いで Arduino UNO から PC へ文字列を送ってみましょう。

仕様は以下の通りです。

仕様:

(1) 電圧レベル変換ICとして ADM3202AN を使う。

(2) データシートでは3.3V駆動になっているが今回は 5V 駆動とする。

(3) T1in と T1out を使う。

(4) PCにはシリアルポートが無いのでUSB-シリアル変換ケーブルを使う。

(5) PC 側の通信アプリとして screen を使う。

(6) 通信速度は 9600 bps とする。

(7) 100ミリ秒毎に Arduino UNO 側から PC 側へ数字がカウントアップしてが送られる( 0, 1, 2, ・・・みたいに)。

データシートや Web 等で回路図を調べて回路を組んで下さい。

　(注意) ジャンパーケーブルが多いと見づらいので
　・1ピン(C1+) - 3ピン(C1-)
　・4ピン(C2+) - 5ピン(C2-)
　・16ピン(Vcc) - 2ピン(V+)
　はコンデンサの足で直結して下さい(ショートさせない様に気を付ける)。

回路を組んだらファシリテータの確認を受けて下さい。
プログラムを書いて Arduino UNO に転送して下さい。

　(注意) Arduino UNO 側のRXピンのジャンパケーブルを外さないとエラーが出ます。

動作したらファシリテータの確認を受けて下さい。

※ screen の起動方法: $ screen /dev/ttyUSB0 9600
※ screen の終了方法: Ctrl 押しながら a を押した後に k 押し、終了するか聞かれたら y

ソースコードをドキュメントにコピーして下さい(回路の写真は必要ありません)。


3. 入力


演習 2-2 (個人PC、30分): Arduino UNO の Rx ピンと PC 側の Tx ピンを繋いで PC から Arduino UNO を制御してみましょう。

仕様は以下の通りです。

仕様:

(1) 回路や通信設定は演習 2-1 とほぼ同じだが、今回は R1in と R1out を使う(T1in と T1outからケーブルを付け替える)。

(2) Arduino UNO の適当な GPIO に LED をつなげる。

(3) PC から 'a' が送られたら LED を点灯する。

(4) PC から 'b' が送られたら LED を消灯する。

仕様に従って回路を修正して下さい。
回路を組んだらファシリテータの確認を受けて下さい。
プログラムを書いて Arduino UNO に転送して下さい。

　(注意) Arduino UNO 側の RX ピンのジャンパケーブルを外さないとエラーが出ます。

動作したらファシリテータの確認を受けて下さい。
ソースコードをドキュメントにコピーして下さい。
回路の全体写真を撮ってドキュメントファイルに貼りつけて下さい。
ドキュメントファイルに「回路の全体写真」と「ソースコード」が記入されていることを確認して下さい。
