アクティビティ: 入出力インターフェース
学習項目: [2] RS-232C
演習時間: 70 分

演習


1. RS-232Cとは

演習 1 (個人PC、10分): シリアルインターフェースについて調べてみましょう。

「RS-232C 以外のシリアルインターフェース」を最低ひとつWeb 等で調べて下さい。
「調べた結果」をドキュメントファイルに記述して保存してください。


2.出力


演習 2 (個人PC、40分): Arduino UNO の Tx ピンと PC 側の Rx ピンを繋いで Arduino UNO から PC へ文字列を送ってみましょう。

仕様は以下の通りです。

仕様:

(1) 電圧レベル変換ICとして ADM3202AN を使う。

(2) T1in と T1out を使う。

(3) PCにはシリアルポートが無いのでUSB-シリアル変換ケーブルを使う。

(4) PC 側の通信アプリとして teraterm を使う。

(5) 100ミリ秒毎に Arduino UNO 側から PC 側へ適当な文字列が送られる。

ホワイトボードに回路図を描いて下さい。

　(注意) データシートでは3.3V駆動になっていますが 5V駆動にして下さい。

回路図を描いたらファシリテータの確認を受けて下さい。
回路図の写真を撮ってドキュメントファイルに貼りつけて下さい。

　(注意) 次の課題でも殆ど同じ回路図を描きますのでホワイトボードは消さないで下さい。

設計図を見ながら回路を組んで下さい。

　(注意) ジャンパーケーブルが多いと見づらいので
　・1ピン(C1+) - 3ピン(C1-)
　・4ピン(C2+) - 5ピン(C2-)
　・16ピン(Vcc) - 2ピン(V+)
　はコンデンサの足で直結して下さい(ショートさせない様に気を付ける)。

回路を組んだらファシリテータの確認を受けて下さい。
回路の全体写真を撮ってドキュメントファイルに貼りつけて下さい。
プログラムを書いて Arduino UNO に転送して下さい。

　(注意) Arduino UNO 側のRX/TXピンのジャンパケーブルを外さないとエラーが出ます。

動作したらファシリテータの確認を受けて下さい。
ソースコードをドキュメントにコピーして下さい。
ドキュメントファイルに「回路図の写真」と「回路の全体写真」と「ソースコード」が記入されていることを確認して下さい。


3.入力


演習 3 (個人PC、20分): Arduino UNO の Rx ピンと PC 側の Tx ピンを繋いで PC から Arduino UNO を制御してみましょう。

仕様は以下の通りです。

仕様:

(1) 回路は演習2とほぼ同じだが、今回は R1in と R1out を使う(T1in と T1outからケーブルを付け替える)。

(2) Arduino UNO の適当な GPIO に LED をつなげる。

(3) PC から 'a' が送られたら LED を点灯する。

(4) PC から 'b' が送られたら LED を消灯する。

ホワイトボードに回路図を描いて下さい。
回路図を描いたらファシリテータの確認を受けて下さい。
回路図の写真を撮ってドキュメントファイルに貼りつけて下さい。
設計図を見ながら回路を修正して下さい。
回路を組んだらファシリテータの確認を受けて下さい。
回路の全体写真を撮ってドキュメントファイルに貼りつけて下さい。
プログラムを書いて Arduino UNO に転送して下さい。

　(注意) Arduino UNO 側のRX/TXピンのジャンパケーブルを外さないとエラーが出ます。

動作したらファシリテータの確認を受けて下さい。
ソースコードをドキュメントにコピーして下さい。
ドキュメントファイルに「回路図の写真」と「回路の全体写真」と「ソースコード」が記入されていることを確認して下さい。
