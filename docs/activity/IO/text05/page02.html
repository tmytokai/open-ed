I2C は SDA / SCL ピンを使ってデータを送受信します。 ここで SDA / SCL ピンの正式名称と用途は以下の通りです。

SDA / SCL ピンの正式名称と用途:

SDA : シリアル・データ ・・・データの送受信に使う

SCL : シリアル・クロック・ ・・・マスタ・スレーブ間の同期を取るためのクロック信号を送る

前述したように I2C はマスタ・スレーブ方式の通信を行うのでマスター側の機器が送受信を管理しています。したがってスレーブ側から送受信を開始することは出来ません。

スレーブ側の機器には個別にアドレスが割り当てられており、まずマスター側が送受信対象となるアドレスを SDA ピン経由で全てのスレーブ側機器に知らせます。 その後指定を受けたスレーブ側機器の準備が整ったら(いわゆるACKを返したら)、SDA ピン経由でマスター側とスレーブ側はデータの送受信を開始します。

なお SCL ピンはデータの送受信には使わず、マスタ・スレーブ間の同期を取るためのクロック信号を送るためのピンとして使われています。

さて、ある機器やマイコン、IC等から別の機器やマイコン、IC等に I2C 通信を使ってデータを送りたい場合は、マスター側とスレーブ側の SDA ピンと SCL ピン(とGND)を線で繋ぎ、メーカーが用意したライブラリの関数を使ってデータを送るだけです。

I2C 通信でデータを送る流れは次のようになっています。なお実際には自分がプログラムしている機器がマスター側なのかスレーブ側なのか考慮しないといけないのですが、今回は簡単のためにマスター側だけを取り扱うことにします。

I2C 通信でデータを送る流れ(マスター側):

(1) 関数を使って送信対象であるスレーブ側機器のアドレスを指定する。

(2) 関数を使ってスレーブ側にデータを送信する。

(3) データを全て送ったら、関数を使ってデータ送信の終了を宣言する。
