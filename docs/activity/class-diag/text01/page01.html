UML(統一モデリング言語)とはオブジェクト指向プログラミングを支援するために作られたモデリング言語のことで、数種類の図(ダイアグラム)の統一的な描き方を定めています。

それらの図の中には「状態遷移図」「シーケンス図」「クラス図」そして「ユースケース図」等が含まれています。
今回はその内、クラス設計で良く用いられる「クラス図」の描き方について学んで行きましょう。

この「クラス図」は「あるクラスと他のクラスの関係を表した図」であり、最も基本となるのはひとつのクラスだけを表した単一クラス図です(図1)。

図1: 単一クラス図

この単一クラス図において

  • 「クラス名」にはそのクラス名
  • 「属性」には各フィールド名
  • 「操作」には各メソッド名

を記入し、更にアクセス修飾子を属性と操作の名前の前に次の記号で記します。

  • public なら「+」
  • private なら 「-」
  • (参考) protected なら「#」

なお、フィールドの型、メソッドの戻り値、メソッドの引数などは省略可能です(見辛くなければあっても構いません)。
またコンストラクタや getter/setter は普通は省略して書きません。
この辺りは特にこうしろという決まりは無いので、自分でやり易いやり方を見つけたり、所属するチームのやり方に従って下さい。

では具体例を挙げてみましょう。例えば次の FRUIT クラス

ソース 1 : 
public class FRUIT{

    private String name;

    public String get_name(){ return name; } // name の getter
    public void set_name( String _name ){ name = _name; } // name の setter

    public FRUIT(){}  // コンストラクタ

    public void eat()
    {
           System.out.println( name + "美味い" );
     }
};

をクラス図で表すと次の様になります。なおフィールドの型、メソッドの戻り値、メソッドの引数、コンストラクタや getter/setter は省略しています。
