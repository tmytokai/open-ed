UMLとクラス図


UML(統一モデリング言語)とはオブジェクト指向プログラミングを支援するために作られたモデリング言語のことで、数種類の図(ダイアグラム)の統一的な描き方を定めています。
それらの図の中には「状態遷移図」「シーケンス図」「クラス図」そして「ユースケース図」等が含まれています。
今回はその内、クラス設計で良く用いられる「クラス図」の描き方について学んで行きましょう。

この「クラス図」は「あるクラスと他のクラスの関係を表した図」であり、最も基本となるのはひとつのクラスだけを表したクラス図です(図1)。 
図1:クラスがひとつだけのクラス図 (クリックで拡大)

この図1において

「クラス名」にはそのクラスの名前

「属性」には各フィールド名

「操作」には各メソッド名

を記入し、更にアクセス修飾子をそれらの名前の前に次の記号で記します。

「+」がpublic

「-」がprivate

「#」がprotected

なお、フィールドの型、メソッドの戻り値、メソッドの引数などは省略可能です。
またコンストラクタや getter/setter は普通は省略して書きません(※)。

※ この辺りは特にこうしろという決まりは無いので、自分でやり易いやり方を見つけたり、所属するチームのやり方に従って下さい。

では具体例を挙げてみましょう。例えば次の HUMAN (人間)クラス 

ソース1:

public class HUMAN{

    private String name;

    public HUMAN(){}

    public void change_name( String new_name ){
          name = new_name; 
    }

    public void show_name(){ 
          System.out.println(name); 
    }
};
