更新内容のプッシュ (衝突が無い場合)


次にプッシュしてローカルリポジトリの更新内容をリモートリポジトリにアップロードして反映します。

ここではリモートリポジトリが更新されていない場合を試します。
この場合はgit によるチーム開発の流れで学んだ「衝突(conflict)」は生じませんので非常にシンプルです。 

 (1) Aさんだけ作業するので、他のメンバーはAさんの作業を後から見る。


 (2) Aさんはプルを実行し、リモートリポジトリの更新内容を
ダウンロードしてローカルリポジトリに取り込む
(ただしこの時点ではリモートリポジトリは更新されていないので 何も起きない)。


git pull


すると「Already up-to-date」と表示される。


 (3) Aさんはプッシュを実行してローカルリポジトリの更新内容をリモートリポジトリにアップロードする


git push


※ IDとパスワードを聞かれますが演習時に知らせます


 (4) Aさんは現在のローカルリポジトリの状態を確認する


git show


 (4) Aさんは現在のリモートリポジトリの状態を確認する


git show origin


※ 前のテキストで説明したように、origin はリモートリポジトリの別名です

すると以下の状態3になります。

AさんのローカルリポジトリのHEADの内容がそのままリモートリポジトリにアップロードされている事が分かります。
