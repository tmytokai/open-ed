既存ファイルの更新


次に既存のファイルを更新してコミットしてみますが、基本的には新規作成と同じ様に add して commit という流れになります。
では実際にやってみましょう。 
 (1) hoge.txtに b という文字を追加して保存する
この時点での「ワーキングツリー」「インデックス」「HEAD」は次のような状態になります。
ここではまだ「ワーキングツリー」しか更新されていません。

A
"piyo.txtを追加" 		← 　
hoge.txt
a
"hoge.txtを追加"

次に前のページで学んだ git add コマンドを使って hoge.txt をインデックスに登録します。 

 (2) hoge.txt が更新されていることを確認: git status


 (3) git add hoge.txt


 (4) hoge.txt がインデックスに登録されているか確認: git status
この時点でワーキングツリーとインデックスは同じ状態になりますが、まだコミットしていませんのでHEADはそのままです。

最後に git commit コマンドを使ってコミットします。 

 (5) git commit -m "hoge.txtを更新"


 (6) コミットするファイルが無いか確認: git status


 (7) コミットログを確認: git log
するとインデックスの内容でスナップショットが作られてローカルリポジトリに登録され、そのスナップショットにHEADが移動します。
