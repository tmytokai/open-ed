「初期設定」


演習 1 (個人、10分): コマンドプロンプトを開いてホームディレクトリ(Z ドライブ)に移動し、git config コマンドで初期設定を行なって下さい。
ユーザーIDとメールアドレスは Office365 に合わせて下さい。


演習 2 (個人、5分): 自分のユーザーIDとメールアドレスが正しく設定されたか git config コマンドで確認して下さい。


演習 3 (個人、5分): type コマンドで .gitconfig の内容を確認してみましょう。

テキストの「状態4」 の状態まで持って行き、git log コマンドでコミットログを確認して下さい。


演習 6 (個人、15分): 同様にテキストの「状態5」 の状態まで持って行き、git log コマンドでコミットログを確認し、コミットが2つ出来たことを確認して下さい。

(※) もし他のメンバーとコミットログの表示が違っている時はどこかで間違えましたので

git reset --hard HEAD~

コマンドを繰り返して「状態4」まで巻き戻してやり直して下さい。
なおこの git reset --hard コマンドは禁じ手なのであまり使わない方が良いです。

「コミットの指定方法」


演習 9 (個人、5分): git show コマンドでコミット情報を表示します。

まず HEAD のコミット情報を表示して下さい。
HEADの 場合は HEAD を省略して git show だけで表示可能です。

次に HEAD ＋チルダ形式を使って 2 つ前のコミットを指定して情報を表示して下さい。


演習 10 (個人、5分): git log で "hoge.txtを更新" というコメントが付いているコミットのリビジョン番号を調べ、 そのリビジョン番号を使ってコミットを指定して情報を表示して下さい。
