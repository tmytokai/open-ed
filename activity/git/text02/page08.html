ファイルの削除


次はファイルの削除方法を学びます。

単にワーキングツリーからファイルを削除しても、そのファイルはインデックスには残り続けるので、何かのタイミングで消したはずのファイルが復活して大問題を起こしたりします。
そこで git で管理しているファイルは次のコマンドを使ってワーキングツリーとインデックスから同時に消す必要があります。 
ワーキングツリーとインデックスからファイルを同時に消す


 git rm <ファイル>
ちなみにインデックスからは消したいけど、ワーキングツリーにはファイルを残したい場合(※)は --cached オプションを付けます。

※ 要するに git による管理を取りやめたい場合です。 

インデックスからファイルを消す(＝ git による管理を止める)


 git rm --cached <ファイル>

では演習として前のページの状態13の状態において piyo.txt を削除してみましょう。
するとワーキングツリーとインデックスの piyo.txt は消えます。ただしまだコミットしていないので HEAD には piyo.txt は残っています。

間違ってファイルを消してしまったとか言う場合は前のページで学んだ git checkout を使って復元できます。
ただしインデックスからも対象のファイルは消えてますので

git checkout HEAD -- <ファイル名>

を使って HEAD から復元する必要があります。

では早速消した piyo.txt を HEAD から復元してみましょう。
