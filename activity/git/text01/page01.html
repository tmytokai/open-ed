バージョン管理システムと git


皆様はソースコードやドキュメント等のファイルのバージョンをきちんと管理していますか？

恐らくほとんどの方は全く管理せず前のファイルに上書きしていったり、していたとしてもせいぜい別名でコピーする位のことしかしていないと思います。
そのため、何か失敗した時にいくつか前の状態に戻したい時は諦めて最初から作り直したり、必死にコピーしたファイルの中身を読み直すという無駄な作業をしている人をよく見かけます。

この様なことをしていると時間の無駄ですので、ソースコードやドキュメント等のファイルは「バージョン管理システム」を使ってバージョンを管理する必要があります。
またこのシステムを使うとクラウド上でファイルのバージョンを一元的管理できますので、チーム開発も容易になります。

さてバージョン管理システムには cvs、svn、git などの無料のものから有料のものまで沢山存在しますが、今回はプログラマーのリーナス・トーバルズが作成し、現在世界で最も良く使用されている git(ギット)の演習を行います。
