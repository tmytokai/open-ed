ローカルリポジトリとリモートリポジトリ

git を使うと、ユーザーはあるファイルを更新した時に、その時点におけるファイルの内容を「リポジトリ」と呼ばれるデータベースに保存できます。
この「リポジトリ」にアクセスすることで、ユーザーはそのファイルを以前の内容に復元したり、編集した箇所の差分を表示したりすることができます。

この「リポジトリ」には「ローカルリポジトリ」と「リモートリポジトリ」の2種類あり、

  • ローカルリポジトリ: 開発者の手元のPC内にあり、その開発者がひとりで利用するためのプライベートなリポジトリ
  • リモートリポジトリ: クラウド上にあり、開発チームに所属する複数の開発者が共同利用するリポジトリ

という違いがあります。

リポジトリをわざわざ「ローカル」と「リモート」の２つに分けている理由は、git は元々世界中に点在する数万人からなるチームのネットワーク上でのソフトウェア開発を念頭にして開発されたためです。
